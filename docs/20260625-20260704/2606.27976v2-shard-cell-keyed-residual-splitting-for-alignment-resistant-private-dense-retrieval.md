## 摘要
稠密嵌入支撑着语义搜索和检索增强生成，然而泄露的向量存储会暴露大部分底层文本。现代的反演和对齐攻击有一个共同弱点：受保护的存储是单一的全局几何结构，而任何单一的几何结构都可以与已知的几何结构对齐——包括秘密的全局旋转，因为正交Procrustes可以从约子空间维度的已知明文对中恢复它。我们提出SHARD，一种保留检索能力的嵌入变换，消除了这一薄弱环节。居中的嵌入被旋转并分割成一个短的公开前缀（驱动第一阶段检索）和一个私有残差，该残差被分割成C个单元，每个单元在独立的密钥下旋转；残差在CKKS下重新排序，其中密钥相互抵消，内积保持精确。一个参数C涵盖了从全局线性基线（C=1）到每文档微密钥（C=N）的范围，使得密钥残差成为文本嵌入的可取消模板——可撤销、可更新、不可链接——这是稠密检索中首个此类方案。在五个编码器上：全维度重排序返回了半SVD截断所放弃的原始空间nDCG@10；在扩散的已知明文泄露下，恢复单元密钥残差需要大约C倍更多的锚点（中位数从C=256时的200到102,400），对于少数加密残差查询，短的公开前缀泄露的邻居结构少得多，微密钥极限将残差泄露降至零。该屏障抵御了学习线性、非线性及无监督对齐器，而在匹配效用的噪声防御中几乎每个探测都被去匿名化时，SHARD完全没有去匿名化。局限性：单元内相似性仍然存在，针对单个受害者单元的目标攻击者仅需约d_priv个锚点，重叠的参考语料库仍会通过公开前缀泄露。SHARD是一种攻击感知的几何防御，而非密码学保证。

## Abstract
Dense embeddings underpin semantic search and retrieval-augmented generation, yet a leaked vector store hands much of the underlying text back. Modern inversion and alignment attacks share one weakness: the protected store is a single global geometry, and any single geometry can be aligned to a known one - a secret global rotation included, since orthogonal Procrustes recovers it from about subspace-dimension known-plaintext pairs. We introduce SHARD, a retrieval-preserving embedding transform that removes that weak axis. The centred embedding is rotated and split into a short public prefix (driving stage-1 retrieval) and a private residual sharded into C cells, each rotated under a separate secret key; the residual is reranked under CKKS, where the keys cancel and the inner product stays exact. One parameter C spans the global-linear baseline (C=1) to per-document micro-keys (C=N), making the keyed residual a cancellable template - revocable, renewable, unlinkable - for text embeddings, the first such scheme for dense retrieval. On five encoders: full-dimensional reranking returns the raw-space nDCG@10 that half-SVD truncation gives up; recovering the cell-keyed residual under a diffuse known-plaintext leak costs about C times more anchors (median 200 to 102,400 at C=256) for a few encrypted residual queries and the short public prefix leaks far less neighbour structure, with a micro-key limit driving residual leakage to zero. The barrier holds against learned-linear, non-linear and unsupervised aligners, and where a matched-utility noise defence de-anonymises almost every probe, SHARD de-anonymises none. Limits: within a cell similarities survive, a targeted attacker on one victim's cell needs only about d_priv anchors, and an overlapping reference corpus still leaks through the public prefix. SHARD is an attack-aware geometric defence, not a cryptographic guarantee.

### 论文提出的方法论：核心思想、关键技术细节

**核心思想**：将嵌入分割为短公共前缀（用于粗粒度检索）和私有残差（用于精排），残差被分片到C个单元，每个单元使用独立秘密正交密钥旋转，在CKKS同态加密下重排序时密钥相互抵消，内积精确恢复，从而将全局几何转化为局部密钥几何。

**关键技术细节**：

1. **离线阶段**：
   - 计算语料嵌入并中心化，通过PCA旋转得到主成分方向。
   - 将每个嵌入分为公共前缀 **u**（前dpub个主成分）和私有残差 **r**（剩余dpriv维）。
   - 根据 **u** 做k-means聚类将文档划分到C个单元。
   - 生成C个独立正交密钥 Hc（Householder反射乘积），每个单元一个。
   - 存储时：单元c(i)中文档i表示为 (ui, zi = Hc(i) ri)。

2. **在线两阶段检索**：
   - **阶段1（公开）**：客户端用公共前缀 u 执行本地ANN（近似最近邻）粗选，得到候选短列表（Kcands个）。
   - **阶段2（加密精排）**：客户端分别加密每个激活单元下的残差（Hc rq）发送给服务器，服务器对每个候选文档执行 ct-pt 内积（zi已存储），由于 Hc 正交性，⟨Hc rq, zi⟩ = ⟨rq, ri⟩，解密后组合公开前缀相似度即得完整余弦相似度 ⟨uq, ui⟩ + ⟨rq, ri⟩ = ⟨xq-μ, xi-μ⟩，实现全维精确重排序。

3. **可取消模板特性**：
   - **不可逆**：无单元密钥无法恢复原生残差。
   - **可撤销与可更新**：用新密钥重新加密残差即可，不影响检索质量。
   - **不可链接**：不同密钥集下的相同文档不可匹配（AUC ≈ 0.5）。
   - **粒度**：C=1退化为全局线性基线；C=N（每文档一密钥）为极端安全端。

### 论文的主要结论与发现

1. **效用恢复**：SHARD全维重排序回收了半SVD截断丢失的原始nDCG@10，在BEIR上对e5-base/SciFact从0.585提升至0.638（与原始0.637无差异），公共前缀仅d/4宽。
2. **抗对齐成本C倍增长**：扩散已知明文下，恢复密钥残差的锚点中位数m50从全局密钥的200升至C=64时的25,600，C=256时达102,400（e5-small）；e5-base下C=256时锚点超过10^5未达50%重识别率。
3. **更强对齐器无法突破**：学习线性（ALGEN核心）、MLP、无监督vec2vec均不优于正交Procrustes（正交Procrustes反而是最强大的）。
4. **公共通道泄露更少**：d/8公共前缀的NN-overlap@10仅0.20~0.55，而基线暴露半空间为0.76；微密钥极限下残差泄露降为0。
5. **高效用与低去匿名化**：在相同效用水平（Acc@1 0.722）下，DP噪声防御去匿名化R@1 ≈ 1.0，而SHARD为0.0。
6. **可取消模板特性**：首次将可取消生物特征模式引入稠密检索，实现零效用损失的可撤销/更新/不可链接模板。

### 优点

- **创新性**：首次将“可取消模板”概念引入文本嵌入隐私保护，提供可撤销、可更新、不可链接的实用机制。
- **实用导向**：参数C连续调节安全性/带宽权衡，从全局基线到微密钥端点，适应不同部署场景。
- **攻击意识设计**：不依赖失真代理（如σrec），而是直接针对对齐和索引泄露攻击面设计。
- **全维精确重排序**：在密钥化残差下仍保持原始空间检索质量，优于截断方法。
- **实验结果严谨**：大量置信区间、配对检验、随机单元对照，提升可信度。
- **透明局限性**：明确列出不保护的内容（单元内相似性、针对受害者单元的攻击、重叠参考查找），体现学术诚实。

### 不足与局限

1. **残差单元内相似性泄露**：同一单元内密钥抵消，服务器可计算精确单元内邻居（C=64时约50%真实邻居可恢复）。
2. **目标攻击可绕过**：若攻击者集中锚点于单一受害者单元，只需约dpriv个锚点即可恢复该单元密钥，与C无关。
3. **重叠参考语料库泄露**：公共前缀仅有一个全局密钥，易被恢复，随后可通过重叠参考语料精确查找文档。
4. **无密码学保证**：仅几何防御，不具备正式隐私保证；文档存储（Erot）在服务器上为明文（仅受代理保护）。
5. **访问模式泄露**：CKKS隐藏查询值和分数，但不隐藏哪些候选文档被重排序；需要额外PIR/ORAM组合。
6. **实验范围限制**：
   - 最大规模为10^6文档，而非典型工业级（亿级）语料。
   - 未在MS MARCO等标准IR基准上测试全端到端性能。
   - 未评估生成式自适应反演（如微调在受保护嵌入上的解码器）。
   - 延迟数据来自本地环路，未考虑分布式网络延迟。
7. **编码器依赖性**：对紧凑/释义编码器（如mpnet, e5-small）需要更宽容的截断，否则效用显著下降。
8. **多语言验证有限**：仅测试两个语言，且语料规模10^5，未覆盖所有主要语种。
