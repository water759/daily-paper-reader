## 摘要
本文提出EXP-SEC这一新颖框架，能够以与安全运营中心分析师领域知识对齐的方式，解释基于深度学习的网络入侵检测系统（导致安全告警）的入侵检测决策。我们强调其以下特点：（1）一个取证模块，可隔离可能引发告警的可疑数据包/流；（2）一个解释模块，能够处理比现有方法更复杂的网络流量特征依赖关系（特征可划分为重叠组，且某些组比其他组更重要）；（3）一个多阶段映射模块，将解释模块生成的基于特征/组的解释转换为适合安全分析师处理的领域特定解释。我们使用最先进的基于深度学习的网络入侵检测系统评估EXP-SEC，结果显示在组级和重叠感知的解释效用指标上，EXP-SEC优于xNIDS（现有最佳性能的解释框架），同时在传统特征级指标（如描述准确性、稀疏性和稳定性）上表现相似。此外，以最先进的基于深度学习的网络入侵检测系统为例，我们展示了EXP-SEC生成的安全分析师友好的解释格式。

## Abstract
In this paper, we present EXP-SEC, a novel framework which can explain the intrusion detection decisions of DL-based NIDS (which lead to security alerts) in a way that is aligned with the domain knowledge of analysts working in Security Operations Center (SOC). We highlight the following features of our framework: (1) a forensic module that isolates the suspect packets/flow which likely caused an alert (2) an explanation module which can handle much more complex feature dependencies in network traffic than existing methods (features can be divided into overlapping groups and some groups are more important than others), and (3) a multi-stage mapping module which translates the feature/group-based explanations generated by explanation module to domain-specific explanations suitable for processing by security analysts. We evaluate EXP-SEC with state-of-the-art DL-based NIDS and our evaluation results show that EXP-SEC outperforms xNIDS (existing best performing explanation framework) in terms of group-level and overlap-aware explanation utility metrics while performing similarly in terms of conventional feature-level metrics such as descriptive accuracy, sparsity and stability. Moreover, taking the case of a state-of-the-art DL-based NIDS, we demonstrate the security analyst-friendly explanation format generated by EXP-SEC.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：基于深度学习的网络入侵检测系统（DL-based NIDS）虽然检测性能优越，但其告警决策缺乏可解释性，导致安全分析师无法理解和信任这些告警，尤其在企业环境中难以有效响应。现有解释方法（如xNIDS）存在明显局限：
  - 无法处理网络流量特征中**重叠的组结构**（同一个特征可能属于多个组，如Kitsune中特征`HpHp_0.1_weight_0`同时属于ARP、TCP、UDP组）。
  - 无法为不同特征组分配**差异化重要性**（某些组比其它组更具预测能力），xNIDS使用的标准SGL对所有组施加相同静态惩罚，导致重要组被过度惩罚或虚假组被错误保留。
- **背景**：网络入侵检测是网络安全的核心，但深度学习模型的黑箱特性使分析师不敢直接响应其告警。现有XAI方法（LIME、SHAP、LEMNA等）在安全域内解释能力不足，xNIDS虽有改进但仍不能处理复杂特征依赖。因此，亟需一个能生成**领域可理解、可操作解释**的框架。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **整体框架 EXP-SEC**：包含三个主要模块：
  1. **取证模块（Forensic Module）**：在NIDS发出告警时，从环形缓冲区的原始数据包中隔离出最可能引起告警的**可疑数据包/流**，并通过全局决策树（DT）近似，筛选出相关窗口包，打包成取证文件供分析师审查。
  2. **解释模块（Explanation Module）**：利用**自适应稀疏组套索（Adaptive Sparse Group Lasso with Overlapping Groups, ASGL-O）** 进行回归优化，生成特征级和组级重要性系数。
     - **特征分组**：根据网络流量结构化特征（如协议类型、流向等）划分特征组，允许特征属于多个组（重叠）。
     - **ASGL-O优化目标**：  
       \[
       \arg\min_{\beta} \left\{ \frac{1}{2n}\|y - g(Z'_{t,m})\|_2^2 + (1-\alpha)\lambda \sum_{q=1}^Q \sqrt{p_q} \tilde{v}_q \|\beta_q\|_2 + \alpha\lambda \sum_{j=1}^p \tilde{w}_j |\beta_j| \right\}
       \]
       其中 \(\tilde{v}_q\) 和 \(\tilde{w}_j\) 是可学习权重，使得重要组/特征被轻罚，无关组/特征被重罚；重叠组通过**潜在变量扩展**处理。
     - 求解采用两阶段估计：先运行初始回归计算自适应权重，再执行稀疏组套索优化，通过内外循环实现组级和特征级稀疏。
  3. **映射模块（Mapping Module）**：将解释模块输出的特征/组系数通过**特征提取框架解构→关联引擎→语义推理引擎**三步，转换为安全分析师能理解的**领域特定解释**（如“高强度机器人扫描”、“持续重数据传输”等）。以Kitsune为例，分解其特征名称（空间上下文、统计量、时间窗口），生成类似“源IP生成大量小包，符合Mirai扫描模式”的自然语言描述。
