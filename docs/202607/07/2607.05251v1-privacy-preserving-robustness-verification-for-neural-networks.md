## 摘要
神经网络验证与数据隐私本质上存在矛盾：验证要求完全访问模型参数和输入数据，但两者都日益受到隐私法规和知识产权限制。这种矛盾使得鲁棒性验证在隐私敏感领域难以实施。在本工作中，我们通过SecureCROWN填补了这一空白，这是首个用于隐私保护神经网络鲁棒性验证的框架。该框架基于安全两方计算（2PC），使模型所有者和数据所有者能够联合计算认证的鲁棒性界限——仅揭示最终结果，同时在半诚实安全模型下可证明地保护双方的私有数据。一个关键挑战是如何安全地计算线性界传播中的条件操作，因为数据依赖的分支与标准安全计算协议不兼容。我们通过将条件逻辑表示为连续算术操作来消除分支。此外，我们引入了牛顿-拉夫森改进方法来提高数值稳定性。广泛的分析和实验表明，SecureCROWN严格匹配明文验证结果，同时在各种模型规模和通信设置（LAN/WAN）下完成时间在0.1-200秒内，证明了隐私保护神经网络验证的可行性。

## Abstract
Neural network verification and data privacy are inherently in tension: verification demands full access to model parameters and input data, yet both are increasingly restricted by privacy regulations and intellectual property constraints. This tension has left robustness verification impractical in privacy-sensitive domains. In this work, we address this gap with SecureCROWN, the first framework for privacy-preserving neural network robustness verification. Built upon secure two-party computation (2PC), our framework enables a model owner and a data owner to jointly compute certified robustness bounds -- revealing only the final result while provably protecting both parties' private data under the semi-honest security model. A key challenge is securely computing the conditional operations in Linear Bound Propagation, where the data-dependent branching is incompatible with standard secure computation protocols. We eliminate branching by formulating conditional logic as continuous arithmetic operations. Additionally, we introduce a Newton--Raphson refinement method to improve numerical stability. Extensive analysis and experiments show that SecureCROWN strictly matches plaintext verification results, while completing in 0.1--200s across varied model sizes and communication settings (LAN/WAN), demonstrating the feasibility of privacy-preserving neural network verification.

### 2. 论文提出的方法论

- **核心思想**：基于安全两方计算（2PC）框架，让模型所有者（P0）和数据所有者（P1）在不泄露各自私有输入（模型参数和输入数据）的情况下，共同计算出认证的鲁棒性边界。协议仅向数据所有者揭示最终的验证结果（是否鲁棒）。
- **关键技术细节**：
    1.  **安全计算框架**：采用 **加法秘密共享（ASS）** 将私有数据（模型参数、输入数据）拆分成随机份额，P0和P1各自持有无法单独解读的份额。所有计算都在这些份额上进行，使用 **Beaver三元组** 进行安全乘法，使用 **函数秘密分享（FSS）** 实现安全比较和非线性函数（如ReLU）。
    2.  **消除条件分支**：标准鲁棒性验证（源于CROWN算法）中，ReLU激活函数的松弛斜率计算依赖于神经元的激活状态（激活、未激活、不稳定），这导致了数据依赖的分支。在MPC中，分支导致昂贵的安全比较。论文提出了一个**无分支的统一算术表达式**来计算松弛斜率 `α`，从而避免了所有分支：
        `α = ReLU(¯z) / (ReLU(¯z) + ReLU(-z) + ϵs)`
        其中，`z`和`¯z`是神经元激活的下界和上界，`ϵs`是防止除零的数值稳定常数。这个公式天然地覆盖了所有三种情况。
    3.  **安全截距累积**：类似地，边界传播中的截距项计算也依赖于系数矩阵的符号。论文使用ReLU操作将条件求和转化为统一矩阵乘法，避免了安全比较。
        - 下界贡献项：`δ = -ReLU(-A) · ReLU(-z)`
        - 上界贡献项：`¯δ = ReLU(A) · ReLU(-z)`
    4.  **数值稳定性优化**：在安全计算除法（求斜率）时，直接使用FSS查找表精度有限。论文引入 **牛顿-拉夫森（Newton-Raphson）迭代** 方法对倒数结果进行精炼，仅需少量迭代即可达到目标精度。
- **算法流程（文字说明）**：
    1.  **离线阶段**：根据公开的网络架构，由可信第三方（或通过协议）生成预处理的随机相关性（Beaver三元组、FSS keys等），此阶段不依赖私有输入。
    2.  **在线阶段**：
        - **输入分发**：双方秘密共享其私有输入（模型参数由P0共享，查询数据由P1共享）。
        - **逐层计算**：对网络从第一层到最后一层逐层进行。对于每一层，调用 `ΠBackward` 协议来安全地计算该层神经元的预激活上下界。然后，利用计算出的上下界，调用 `Πα` 协议（使用上述无分支公式）计算出该层ReLU的松弛斜率。
        - **最终验证**：在获得所有层的松弛斜率后，针对目标验证类别对（y, j），使用差异向量 `d` 初始化，再次调用 `ΠBackward` 协议，计算出最终的认证下界 `fy,j`。
        - **结果输出**：P0将自己的份额发送给P1，P1恢复出 `fy,j`。如果 `fy,j > 0`，则证明该模型在给定输入和扰动范围内是鲁棒的。
