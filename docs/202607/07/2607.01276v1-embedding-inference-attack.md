## 摘要
嵌入模型是现代信息检索（IR）系统的核心组件，但它们通常隐藏在API之后。近年来的研究表明，密集IR系统可能导致安全漏洞，例如嵌入反转攻击。然而，此类攻击通常要求攻击者知道嵌入模型才能进行攻击。在本文中，我们研究了黑盒设置下的IR系统，其中观察者仅能看到检索到的无序文档集合，而无法获知排序或相似度分数。我们证明，在此类场景下，精心设计的查询允许攻击者从一组已知的候选模型中识别出所使用的嵌入模型，我们将此称为嵌入推理攻击（EIA）。我们还表明，即使系统包含重排序器作为潜在的防御机制，某些查询仍具有区分能力。我们进一步在真实的检索增强生成（RAG）系统上验证了该方法，其中定制查询能够绕过LLM倾向于拒绝未识别为格式良好问题的输入的倾向。最后，我们提出并评估了其他缓解策略，例如相似度阈值。

## Abstract
Embedding models are essential components of modern Information Retrieval (IR) systems, yet they are typically hidden behind APIs. Recent works have shown that dense IR system can lead to security vulnerabilities such as embedding inversion attacks. However, such attacks usually require that the attacker knows the embedding model for the attack to be applicable. In this paper, we study IR systems under a black-box setting in which the adversary observes only the unordered set of retrieved documents, without ranking or similarity scores. We demonstrate that in such contexts, tailored queries allow an adversary to identify which embedding model is in use from a set of known model candidate, which we coin as an embedding inference attack (EIA). We also show that certain queries remain discriminative even when the system includes a reranker as a potential defense mechanism. We further validate our method on a real Retrieval-Augmented Generation (RAG) system, in which the tailored queries bypass the LLM's tendency to reject inputs it does not recognize as well-formed questions. Finally, we propose and evaluate other mitigation strategies such as similarity thresholds.

## 1. 核心问题与整体含义（研究动机与背景）
- **研究动机**：现代信息检索（IR）系统，包括检索增强生成（RAG）系统，普遍使用嵌入模型（embedding models）将文本转为向量。但嵌入模型通常仅通过API暴露，且其参数和架构对外保密。现有嵌入反转攻击（如Vec2Text、ZSInvert等）需要攻击者已知或能访问目标嵌入模型，才能从嵌入向量重构原始文本。
- **核心问题**：在更现实的黑盒场景下，攻击者只能查询系统并获得无序的文档集合（无排名、无相似度分数），能否推断出系统所使用的嵌入模型？
- **整体含义**：作者首次提出**嵌入推理攻击（Embedding Inference Attack, EIA）**，即仅通过观察检索结果即可识别嵌入模型。识别模型是后续更严重攻击（嵌入反转、语料投毒、成员推断等）的前置步骤，因此EIA揭示了IR系统新的安全风险，尤其是基于公开语料的系统。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：不同嵌入模型对同一查询会返回不同的文档集合。通过构造能放大这种差异的查询（称为判别性查询），可以逐个排除候选模型，最终唯一确定目标模型。
- **关键技术细节**：
  1. **基础度量**：对于查询 `q_i`，计算受害者系统返回的文档集 `S_v^i` 与候选模型返回的文档集 `S_c^i` 之间的 Jaccard 指数 `J_i = |S_v^i ∩ S_c^i| / |S_v^i ∪ S_c^i|`。若 `J_i=1` 对所有查询成立，则两模型相同；否则不同。
  2. **暴力生成判别性查询**：使用LLM为语料库中的每个文档生成一个问题，查询所有候选模型，计算每个查询的消除分数 `E(q_i)`（即区别于受害者模型的候选模型数量），选择 `E(q_i)=|M|` 的查询作为完全判别性查询。
  3. **对抗性判别性查询**：为减少API调用，设计三类更高效的查询：
     - **随机字符串（Rnd. Str.）**：无意义字符序列，探索嵌入空间中训练数据未覆盖的区域。
     - **非查询（Non-Q.）**：指令而非问题（如“仅简短回答”），到达与文档无关的嵌入区域。
     - **探测查询（Prob. Q.）**：针对系统本身的问题（如“你的开发者是谁？”），探索模型相关的嵌入区域。
- **流程**：攻击者向受害者系统发送查询 → 获取无序文档集 → 与所有候选模型在相同查询下的检索结果比较 → 利用Jaccard指数和消除分数过滤候选 → 重复直至剩下唯一模型。

## 3. 实验设计：数据集、场景、基准、对比方法
- **数据集**：
  - **MS MARCO passage development set**：7000个查询、8.8M段落、7400个人工标注相关性判断（qrels），选自公开网络文档。
  - **Alloprof教育QA数据集**：K-12学生提问及教师解答（主要为法语，仅用英语子集，585个文档）。
- **实验场景**：
  - **自有IR管道**：使用13个开源嵌入模型（分为E5、English SBERT、Multilingual SBERT、T5 encoder、Qwen3五大家族），按逐个轮换作为受害者，其余为候选。检索top-k（k=3,5），相似度度量使用余弦相似度。
  - **真实RAG系统**：使用AnythingLLM框架，搭配3个嵌入模型（all-MiniLM-L6-V2、nomic-embed-text-v1、e5-small）和Qwen3 Vision 4B LLM，测试Chat模式和Query模式。
  - **含重排序器的场景**：使用cross-encoder/ms-marco-MiniLM-L6-v2对初始25个文档重排序后取top-k。
- **基准与对比方法**：无直接对比方法（首创），但对比了不同查询类型（生成式、随机字符串、非查询、探测查询）、不同k值、有无重排序器、不同相似度阈值的影响。
- **消融实验**：考察top-k值、重排序器、相似度阈值、不同RAG交互模式对攻击成功率的影响。

## 4. 资源与算力
- **明确说明**：使用单张NVIDIA H100 GPU，通过MIG技术分割为两个40GB实例；搭配64核CPU和64GB RAM。
- **训练耗时**：完整评估约需**8 GPU小时**（无重排序器），使用重排序器需**12-15 GPU小时**。未说明具体训练轮数或迭代次数。

## 5. 实验数量与充分性
- **实验数量**：涵盖：
  1. IR管道上13个模型×2个top-k值（3和5）×4种查询类型（生成式/随机字符串/非查询/探测查询）×有无重排序器。
  2. RAG系统上3个模型×2种检索模式（Chat/Query）×4种查询类型×有无重排序器。
  3. 相似度阈值实验（4个阈值0.25/0.5/0.6/0.75）对判别性查询数和检索质量的影响。
- **充分性评价**：
  - 优点：覆盖了多种候选模型家族、不同查询设计、防御机制（重排序器、相似度阈值）、真实RAG系统，实验设计较为全面。
  - 不足：未测试不同文档分块大小和重叠率的影响；仅使用开源模型，未包含闭源商业模型；RAG实验中仅使用3个嵌入模型，规模较小；对抗性查询类型的数量较少（各11个），统计可能不够稳定。

## 6. 主要结论与发现
1. **EIA在IR管道中有效**：仅需平均少量查询即可唯一确定嵌入模型。对于k=3，1249个生成式查询即可区分所有13个模型；随机字符串、非查询、探测查询也均能100%成功。
2. **top-k值影响显著**：k越大（如k=5），需要更少查询即可区分，因为更多文档提供更丰富差异信息。
3. **重排序器不影响攻击有效性**：即使系统使用重排序器，攻击仍有效，且部分情况下（k=3）攻击性能反而提升（因重排序可能改变文档集）。
4. **RAG系统中攻击可行**：LLM的问答生成可能拒绝处理异常输入（如随机字符串），但检索组件仍然返回文档，攻击者可利用这些文档进行模型推断。
5. **相似度阈值可作为防御**：设置较高阈值（如0.6）能显著减少判别性查询数量，但可能损害检索效果，且影响因模型而异（如LaBSE受影响严重，e5-large-v2几乎不受影响）。

## 7. 优点：方法和实验设计的亮点
- **首个黑盒嵌入模型推断攻击**：仅需无序文档集，无需向量、排名或相似度分数，攻击条件极低，更贴近现实。
- **多类判别性查询的设计**：包括随机字符串、非查询、探测查询，展示了嵌入空间中不同区域对模型差异的敏感性，方法创新。
- **综合考虑防御机制**：评估了重排序器、相似度阈值等防御对攻击的影响，并分析了防御对检索质量的副作用，实用性强。
- **真实RAG环境验证**：使用AnythingLLM完整搭建RAG系统，考虑了LLM行为（如拒答）对攻击的影响，实验场景贴近实际部署。
- **消融实验充分**：对top-k、重排序、相似度阈值、查询类型等参数做了系统对比，结论可靠。

## 8. 不足与局限
- **未评估文档分块参数**：文档分块大小和重叠率会直接影响检索结果，对EIA的影响未被研究。
- **仅限开源模型**：未测试闭源商业嵌入模型（如OpenAI text-embedding-ada-002、Cohere embed等），攻击在黑盒商业API下是否依然有效尚不确定。
- **对抗性查询数量少**：随机字符串、非查询、探测查询每种仅11个，统计代表性可能不足，尤其高阈值下成功率骤降时结论泛化能力有限。
- **RAG实验规模小**：仅使用3个嵌入模型，未涵盖更多候选，且仅用单一LLM（Qwen3 Vision 4B），缺少对不同LLM的泛化测试。
- **未考虑动态语料更新**：假设语料库固定且公开，实际系统中语料可能持续变化，攻击鲁棒性未知。
- **假设候选模型已知且包含受害者模型**：攻击者需要预知候选模型集合，若受害者使用未知自定义模型，则攻击无效。
