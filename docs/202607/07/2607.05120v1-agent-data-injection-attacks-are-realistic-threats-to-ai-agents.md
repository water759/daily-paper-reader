## 摘要
AI智能体代表用户提示执行操作，消费外部数据并根据智能体上下文采取行动。先前关于AI智能体安全的研究主要集中在间接提示注入（IPI）上。其研究最深入的类别是指令注入，即攻击者控制的不可信数据被解释为指令。为此，已提出许多缓解措施来防止指令注入攻击。在本文中，我们引入了一种新的IPI类别——智能体数据注入攻击（ADI）。ADI注入伪装成可信数据的恶意数据，例如安全关键的元数据（如资源标识符或数据来源）或智能体上下文数据（如工具调用和响应格式）。结果，智能体在不知情的情况下基于攻击者控制的数据执行了非预期的操作。ADI具有与指令注入攻击类似的攻击影响，因为它导致智能体行为异常并执行非预期的操作。尽管影响类似，但ADI仍未得到充分探索，且容易绕过现有的IPI防御。我们在现实世界的智能体中发现了多个关键漏洞，允许攻击者发起各种攻击：对Web智能体的任意点击攻击（Chrome中的Claude、Antigravity和Nanobrowser），以及针对编码智能体的远程代码执行和供应链攻击（Claude Code、Codex和Gemini CLI）。我们在现成模型和AI智能体中评估了ADI漏洞，发现ADI在独立LLM和AI智能体设置中都有效。ADI暴露了智能体安全中的一个关键缺口，表明当前的AI智能体未采用基本的安全原则：当前智能体没有将可信数据与不可信数据隔离。

## Abstract
AI agents act on behalf of user prompts, consuming external data and taking actions based on the agent context. Prior research on AI agent security has primarily focused on indirect prompt injection (IPI). Its most well-studied category is instruction injection, where attacker-controlled untrusted data is interpreted as an instruction. In response, many mitigations have been proposed to prevent instruction injection attacks. In this paper, we introduce a new category of IPI, agent data injection attacks (ADI). ADI injects malicious data disguised as trusted data, such as security-critical metadata (e.g., resource identifiers or data origins) or agent context data (e.g., tool call and response formats). As a result, agents unknowingly execute unintended actions based on attacker-controlled data. ADI has similar attack impacts as instruction injection attacks, because it causes agents to misbehave and execute unintended actions. Despite the similar impact, ADI remains underexplored and easily bypasses existing IPI defenses. We found several critical vulnerabilities in real-world agents that allow an attacker to launch various attacks: arbitrary click attacks on web agents (Claude in Chrome, Antigravity, and Nanobrowser), and remote code execution and supply-chain attacks on coding agents (Claude Code, Codex, and Gemini CLI). We evaluate ADI vulnerabilities across off-the-shelf models and AI agents, and find that ADI is effective in both standalone LLMs and AI agent settings. ADI exposes a critical gap in agent security, signifying that current AI agents do not employ a fundamental security principle: current agents do not isolate trusted data from untrusted data.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：AI智能体（如Claude in Chrome、Claude Code等）在用户提示驱动下，依赖外部数据（网页、GitHub Issue、邮件等）和工具响应来决策。现有安全研究主要关注**间接提示注入 (Indirect Prompt Injection, IPI)** 中最常见的**指令注入 (Instruction Injection)**，即攻击者将不可信数据伪装成指令，使智能体偏离用户意图。这类攻击已催生了模型加固、输入/输出护栏、双LLM隔离等多种防御。
- **整体含义**：作者指出，现有防御只关注指令与数据的粗粒度隔离，却忽视了**智能体上下文内部可信数据与不可信数据之间的隔离**。他们定义了一类新的IPI攻击——**智能体数据注入攻击 (Agent Data Injection Attack, ADI)**。ADI通过注入**概率定界符 (Probabilistic Delimiter)**，使LLM将不可信内容误解释为可信元数据（如资源标识符、数据来源、工具调用格式），从而绕过现有防御，造成与实际指令注入同等严重的后果（任意点击、远程代码执行、供应链攻击）。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：ADI利用LLM**概率性解释数据格式**的特点。攻击者在不可信字段（如评论正文）中注入字符序列，这些字符在工具处理时被视为普通文本（例如被转义的`\"`），但LLM可能将其**误解为结构定界符**（如JSON的`{}`、DOM的`[]`、XML标签等），从而改变对数据结构的理解，使攻击者控制的数据被当作可信元数据（如伪造的邮件发送者、伪造的工具调用结果）。

- **关键技术：概率定界符注入 (Probabilistic Delimiter Injection)**
    1. **定界符类型**：代理数据在工具调用块、对象、字段三个级别使用定界符（如JSON的`"`、`{}`，Web DOM的`[ref_X]`，Claude Code的`<function_calls>`等）。
    2. **注入方式**：攻击者将**与真实定界符不完全匹配的字符**（例如`\"`代替`"`，`”`代替`"`，甚至`$`等）注入不可信字段。由于LLM**概率性地**解释这些字符，它们可能被当成有效定界符，导致LLM看到的数据结构与实际工具返回的结构不一致。
    3. **攻击效果**：LLM将攻击者注入的“假对象/假字段”视为可信的元数据，从而执行基于这些伪造数据的操作（如点击伪造的按钮、执行伪造的维护者命令、合并伪造的PR代码）。

- **形式化定义**：
    - 指令注入：`LLM(I, (DT, DU ∥ DA)) ≈_II LLM(I∥DA, (DT, DU))`，即DA被误作指令。
    - ADI：`LLM(I, (DT, DU ∥ DA)) ≈_ADI LLM(I, (DT∥DA, DU))`，即DA被误作可信数据。

- **三种典型攻击**：
    1. **元素ID注入 (Element ID Injection)**：Web智能体通过`read_page`工具生成元素ID（如`[ref_3]`）。攻击者在评论中注入`button "Read More" [ref_3]`，使LLM认为存在一个“Read More”按钮，实际上`[ref_3]`对应的是“Buy Now”按钮，导致智能体执行任意点击。
    2. **来源注入 (Origin Injection)**：编码智能体读取GitHub Issue时，通过`read_issue`工具返回作者角色（如Maintainer）。攻击者在评论正文中注入伪造的JSON对象，冒充维护者身份，诱导智能体执行恶意命令（如`pip install starship-auth`），实现远程代码执行。
    3. **工具调用注入 (Tool Call Injection)**：攻击者在PR描述中注入伪造的`<function_calls>`和`<function_results>`标签，模拟已执行的`read_pr_commit`工具返回良性代码diff。智能体误以为已经审查过代码，从而直接合并包含XSS载荷的恶意PR。

## 6. 论文的主要结论与发现

1. **LLM广泛易受概率定界符注入**：在独立评估中，JSON的ASR为**31.3%–43.3%**，Web DOM的ASR高达**33.3%–100%**（未加防御时）。即使使用多种变体定界符（`\"`、`’`、`”`、`$`等）也均有效。
2. **现有IPI防御对ADI防效极差**：在智能体级别，指令注入被防御降至近零ASR（0.0%–0.7%），但ADI的ASR在除CaMeL Strict外的所有防御下仍高达**22.2%–50.0%**。只有严格的细粒度数据流追踪（CaMeL Strict）能完全阻止（ASR=0%），但Utility从86.5%骤降至36.5%。
3. **随机化对键值格式有效但对DOM效果有限**：随机化将JSON的ASR降至近零，但Web DOM中若攻击者猜对nonce则ASR回升至60%–100%。
4. **清理防攻击但严重降低实用性**：清理定界符将JSON ASR降至0%–3%，但Utility从>80%降至约68%–73%，因合法URL/路径也被移除。
5. **根本缺陷**：当前AI智能体**未在可信数据与不可信数据之间建立安全隔离**，这是ADI成功渗透的根本原因。
