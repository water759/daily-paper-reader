## 摘要
全同态加密（FHE）支持在加密数据上进行计算，但实际的加密Transformer推理受到许多非线性模块顺序组合的瓶颈限制。我们研究结构化牛顿层级并行（SNLP）是否能使这种层间组合更加FHE友好：每个Transformer块仍然需要针对softmax和RMSNorm等操作的多项式逼近，但SNLP将层级顺序非线性深度从L阶段减少到少量求解器迭代加上线性结构化校正。使用基于切比雪夫多项式逼近的模拟框架，我们测量了8个模型和4个架构系列在顺序推理与SNLP推理下的误差累积。在0.5B IDN训练的模型上，SNLP将符号引导次数从53次减少到20次（2.65倍），困惑度仅增加1.2%，同时降低了误差放大（1.36倍对比1.42倍）。在所有测试模型中，SNLP的放大率均低于顺序推理。消融实验表明，softmax逼近主导误差预算，而CKKS算术噪声在我们的设置中可忽略不计，这表明SNLP是对块级FHE友好算子设计的补充，而非替代。

## Abstract
Fully homomorphic encryption (FHE) enables computation on encrypted data, but practical encrypted Transformer inference is bottlenecked by the sequential composition of many nonlinear blocks. We study whether Structured Newton Layer Parallelism (SNLP) can make this inter-layer composition more FHE-friendly: each Transformer block still requires polynomial approximations for operations such as softmax and RMSNorm, but SNLP reduces the layerwise sequential nonlinear depth from L stages to a small number of solver iterations plus linear structured corrections. Using a simulation framework based on Chebyshev polynomial approximations, we measure error accumulation under sequential versus SNLP inference across 8 models and 4 architecture families. On a 0.5B IDN-trained model, SNLP reduces symbolic bootstraps from 53 to 20 (2.65x) with only +1.2% perplexity degradation, while lowering error amplification (1.36x vs. 1.42x). Across all tested models, SNLP has lower amplification than sequential inference. Ablations show that softmax approximation dominates the error budget and CKKS arithmetic noise is negligible in our setting, suggesting that SNLP is complementary to block-level FHE-friendly operator design rather than a replacement for it.

- **核心问题**：能否通过改变Transformer层间计算图（而非改造单个块内的算子）来降低加密非线性深度，从而减少自举次数并控制误差放大？
- **整体含义**：本文证明结构化牛顿层并行（SNLP）是一种互补于块级FHE友好算子设计的策略，它通过将L层顺序非线性深度压缩为少量迭代加线性校正，显著降低FHE推理成本，同时保持较低的精度损失和误差放大。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 将Transformer层的正向传播视为一个非线性残差方程组，通过结构化牛顿迭代并行求解后缀层，而非逐层顺序计算。
- 每个非线性块内部的算子仍需多项式近似，但层间组合从“L个顺序非线性阶段”变为“少量迭代 + 纯线性校正”，从而减少加密非线性深度。

### 关键技术细节
1. **问题建模**：将隐藏状态序列 `h = (h1,...,hL)` 视为残差方程 `Gl(h) = hl - fl(hl-1) = 0` 的解，其中 `fl` 为第l个Transformer块。
2. **SNLP流程**：
   - **前缀**：顺序计算前S层，得到 `hS`。
   - **后缀**：剩余 `N = L - S` 层并行求解，经过K次迭代：
     - 并行评估：`\tilde{h}_l^{(k)} = f_l(h_{l-1}^{(k)})`（所有后缀层独立计算，可批处理）
     - 结构化牛顿校正：`h_l^{(k+1)} = \tilde{h}_l^{(k)} + A_l^{(k)} (h_{l-1}^{(k+1)} - h_{l-1}^{(k)})`，其中 `A_l^{(k)}` 是块雅可比矩阵的廉价结构化近似。
3. **两种结构化近似**：
   - **IDN（Identity Newton）**：`A_l = I`，校正变为纯加法前缀和传播，FHE乘法深度为零。
   - **HCN（HC Newton）**：使用mHC残差混合矩阵 `A_l = H_l^{res,mlp} H_l^{res,attn}`（小S×S矩阵），仅需线性乘加。
4. **SNLP感知训练**：在预训练时增加辅助损失，使有限迭代的SNLP求解结果逼近顺序轨迹，从而适配廉价近似。
5. **FHE成本代理指标**：定义 `NFE = (L - N) + K`（非线性正向评估次数）。在符号CKKS成本模型中，NFE与自举次数呈线性关系（比例0.99-1.02×），因为IDN校正不消耗乘法深度。

## 6. 主要结论与发现

1. **SNLP始终优于顺序推理**：在所有8个模型、所有测试多项式度数下，SNLP的误差放大均低于顺序推理。0.5B IDN模型上，SNLP n24-K4将自举从53降至20（2.65×），PPL仅增加1.2%，误差放大1.36× vs. 1.42×。
2. **mHC架构最FHE友好**：在0.5B模型中，mHC架构顺序推理的误差放大率最低（1.24×），接近3B模型（1.25×），因其超连接为纯线性操作，不增加非线性深度。
3. **FHE最优配置 ≠ 墙钟最优配置**：SNLP n24-K8（K=8）在GPU上比顺序推理慢，但在FHE中因深度浅、自举少而更优，说明存在独立的FHE优化前沿。
4. **softmax近似是主要误差源**：消融实验中，softmax单独近似导致PPL增加43.7%，其他操作影响<13%。CKKS算术噪声在测试精度下可忽略。
5. **SNLP感知训练至关重要**：IDN训练的模型才能在低NFE区间（12-16）保持高质量，基线模型无法低于NFE=20。
6. **更大模型更鲁棒**：3B模型误差放大（1.25×）低于0.5B（1.42×），高维隐藏状态稀释逐元素近似误差。

- **指标设计合理**：提出NFE作为FHE成本的轻量代理，与自举次数高度相关，便于在模拟中快速评估。
