## 摘要
侧信道攻击对现代计算平台构成严重安全威胁，因为它们利用CPU行为的细微差异泄露敏感信息。为了建模CPU通过微架构侧信道泄露的信息，近期工作提出了泄露契约：一种ISA级安全抽象，为安全CPU编程提供基础。不幸的是，由于当前微架构的复杂性，为CPU设计泄露契约需要大量手动工作，因此现代CPU缺乏专用的泄露契约。我们提出了一种方法，以最少的手动干预为主要CPU架构提取指令中心泄露契约。我们在malcos中实现了该技术，这是首个用于黑盒CPU泄露契约综合的无模板自动化工具。我们在x86和ARM CPU上评估了malcos，并展示其综合的契约相对于综合过程中观察到的所有泄露是精确且合理的。我们的结果表明，从黑盒CPU学习泄露契约是可行的。

## Abstract
Side-channel attacks pose a significant security threat for modern computing platforms, because they exploit subtle discrepancies in CPU behaviors to leak sensitive information. To model the information leaked by a CPU via microarchitectural side-channels, recent work proposed leakage contracts: an ISA-level security abstraction that provides the foundations for secure CPU programming. Unfortunately, due to the complexity of current microarchitectures, devising a leakage contract for a CPU requires extensive manual effort and thus modern CPUs lack dedicated leakage contracts.   We present a methodology to extract instruction-centric leakage contracts for major CPU architectures with minimal manual intervention. We implemented this technique in malcos, the first template-free tool that automates the synthesis of leakage contracts for black-box CPUs. We evaluate malcos on x86 and ARM CPUs, and show that the contracts it synthesizes are precise and sound with respect to all leaks observed during synthesis. Our results demonstrate that learning leakage contracts from black-box CPUs is feasible.

## 1. 论文的核心问题与整体含义（研究动机和背景）

侧信道攻击利用CPU微架构行为的细微差异泄露敏感信息（如执行时间、缓存访问模式）。为了建模这些泄露，研究者提出了**泄露契约（Leakage Contracts）**：一种ISA级安全抽象，为安全编程提供基础。然而，由于现代CPU微架构的复杂性，手工构建泄露契约需要大量专家知识和时间，导致现有CPU普遍缺乏专用契约。已有的自动化综合工具（如RTL2μPath、LeaSyn等）存在两大局限：①需要访问CPU的寄存器传输级（RTL）设计，仅适用于小规模RISC-V核心，无法用于商业x86/ARM CPU；②依赖用户提供的模板，限制了可捕获的泄露类型（例如无法覆盖“操作数为0或1”导致的时序泄露）。

本文提出**malcos**——首个**无模板、黑盒**的指令中心泄露契约自动综合工具，能够从商业CPU中学习精确且完备的契约，大幅减少人工干预，为安全编程提供自动化基础。

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

### 核心思想
采用**反例驱动综合（CEGIS）** 方法，交替进行两步：
- **Step 1（泄漏测试）**：基于当前候选契约，使用关系型泄漏测试工具（Revizor for x86，Scam-V for ARM）生成测试程序并在黑盒CPU上运行，发现未被捕获的新泄露（反例cex）以及正例pex（不可区分执行的例子）。
- **Step 2（契约精炼）**：使用Rosette求解器将新发现的泄露综合成新的ISA级契约子句，确保候选契约捕获该泄露。通过反例保证完备性，正例保证精确性（避免过度近似）。

### 关键技术细节
- **icl语言**：定义了契约子句语法 `ex IF pr`，其中`ex`是泄露表达式，`pr`是触发条件。支持操作数类型、访问模式、操作数值等。
- **契约精炼**：将每个反例对应到程序中的具体指令，通过合成满足硬约束（必须区分反例）和软约束（最大化区分反例、最小化区分正例）的子句，然后**泛化**为与指令类型匹配的通用子句（替换程序计数器为操作码和操作数类型条件）。
- **最小化**：后处理阶段，按精确性排序子句，迭代移除那些不影响区分所有反例的子句，提升契约精度。

### 算法流程（Algorithm 1）
1. 生成`n`个种子程序。
2. 对每个程序，初始化候选契约（可基于之前结果或重置）。
3. 循环：调用Checker获取反例和正例；若有反例，则调用Refiner合成新子句加入候选；若无则退出。
4. 合并所有子句，进行最小化，输出最终契约。

1. **malcos能精确学习合约**：在所有模拟场景中，malcos合成的合约达到100%完备性（soundness），且在大多数情况达到100%精确性（precision）。TagIdx场景因低质量正例导致精度79%，可通过调整重置参数提升至96%。

5. **实用性证据**：malcos可在合理时间内（每个CPU约20小时）完成对商业CPU的合约综合，且无需RTL或模板。

## 7. 优点

- **彻底无模板**：消除了限制性模板，能发现模板遗漏的复杂泄露（如乘法简化、寄存器文件压缩）。
- **黑盒适用**：无需RTL设计，可直接用于商业CPU，具有广泛实用性。
- **双重保证**：通过反例保证完备性（捕获所有发现的泄露），通过正例保证精度，避免过度近似。
- **模块化设计**：Checker和Refiner可替换，支持不同架构和不同观测模型（时序、缓存、未来可扩展至功耗等）。
- **后处理最小化**：自动去除冗余子句，提升合约可读性和精确性。

## 8. 不足与局限

- **仅限指令中心合约**：不能表示跨多条指令的状态性泄露（如计算重用、操作数打包），也无法处理内核级泄露（如页表访问）。
- **有界完备性**：仅保证捕捉测试过程中暴露的所有泄露，未暴露的泄露（即假阴性）可能遗漏，依赖于Checker的覆盖率。
- **精度依赖测试质量**：低质量正例会导致过度近似（如TagIdx的79%精度），需要调参（重置频率、正例数量）缓解。
- **权限限制**：仅支持用户态代码，无法处理特权指令或异常处理中的泄露。
- **硬件差异**：实验仅覆盖有限CPU型号（4款Intel、2款ARM），未在AMD或其他ARM核心上验证，通用性有待进一步证明。
- **无硬件RTL验证**：相比于白盒方法，malcos不能从设计根源保证完备性，仅基于行为观测。
