## 摘要
自主人工智能代理越来越多地在人类有限监督下执行行动、调用工具和操作受保护资源。现有的认证和授权机制建立身份并委派权限，但本质上不提供加密证据来证明由特定代理发出的具体请求在特定执行上下文中满足适用策略。本文假设代理授权可以形式化为一个可加密验证的关系，记为$R_{CVA}$，该关系联合绑定代理主体、具体授权请求、执行上下文以及适用策略的满足情况，同时有选择地保护私有授权属性的机密性。我们介绍了可加密验证代理授权（CVA）的初步形式化抽象，定义了一组紧凑的候选安全属性，包括授权健全性、主体绑定、请求绑定、策略绑定和重放抵抗，并提供了一个可执行的零知识概念验证，该验证在Groth16 zk-SNARK构造上实例化了模型的选定元素。我们进一步识别并形式化了身份绑定、授权请求绑定和运行时执行绑定之间的结构分离，将其作为安全代理系统设计中的一个核心开放问题（当前代理安全框架未明确解决的区别），并提出了一个可证伪的研究议程来解决该问题。

## Abstract
Autonomous AI agents increasingly execute actions, invoke tools, and operate on protected resources with limited human oversight. Existing authentication and authorization mechanisms establish identity and delegate authority, but do not inherently provide cryptographic evidence that a concrete request issued by a specific agent satisfies the applicable policy in a specific execution context. This paper hypothesizes that agent authorization can be formalized as a cryptographically verifiable relation, denoted $R_{CVA}$, that jointly binds an agent principal, a concrete authorization request, an execution context, and the satisfaction of an applicable policy, while selectively preserving the confidentiality of private authorization attributes. We introduce a preliminary formal abstraction for Cryptographically Verifiable Agent Authorization (CVA), define a compact set of candidate security properties including authorization soundness, principal binding, request binding, policy binding, and replay resistance, and provide an executable zero-knowledge proof of concept that instantiates selected elements of the model over a Groth16 zk-SNARK construction. We further identify and formalize the structural separation among identity binding, authorization-request binding, and runtime execution binding as a central open problem in the design of secure agentic systems (a distinction {not explicitly addressed by} current agentic security frameworks) and present a falsifiable research agenda for its resolution.

### 核心思想
- **CVA假设（H1）**：授权决策可以表示为同时绑定代理主体、具体授权请求、执行上下文和策略满足的加密可验证关系，同时选择性地保护私有授权属性的机密性。
- 定义关系 \( R_{CVA}(x, w) \in \{0,1\} \)，其中：
  - 公共语句 \( x = (id_i, h_q, h_c, pid_j, n, t) \)：包括代理ID、请求承诺、上下文承诺、策略ID、nonce、时间戳。
  - 私有见证 \( w = (sk_i, \rho_i, attrs_i, q_{priv}, c_{priv}) \)：包括代理密钥、随机数、私有属性、请求明文、上下文明文。
- 证明系统：\( \pi \leftarrow \text{Prove}(pp, x, w) \)，验证者调用 \( \text{VerifyAuth}(pp, x, \pi) = 1 \) 接受。

### 关键技术细节
- 关系分解为四个合取：`BindPrincipal ∧ BindRequest ∧ BindContext ∧ SatisfyPolicy`。
  - `BindPrincipal`：`CommitID(sk_i, ρ_i) = id_i`，使用承诺方案（原型中简化为Poseidon哈希）。
  - `BindRequest`：`Hq(Encode(q_{priv})) = h_q`，使用碰撞抵抗哈希。
  - `BindContext`：`Hc(Encode(c_{priv})) = h_c`，哈希承诺上下文。
  - `SatisfyPolicy`：`Pj(attrs_i, q_{priv}, c_{priv}) = 1`，策略作为确定性谓词。
- 使用**Groth16 zk-SNARK**作为底层证明系统，选择bn128曲线，电路写于Circom 2.x，证明生成用snarkjs，网关用FastAPI实现。
- 新鲜性（重放抵抗）由网关外部维护nonce集合 \( N \)，分离于无状态证明验证。

### 威胁模型
- 对手能观察、拦截、重放证明，替换标识符、修改承诺，但不能破坏加密基元或获取诚实方秘密。
- 攻击类别：证明伪造、跨主体转移、跨请求转移、跨策略转移、重放。

## 6. 论文的主要结论与发现

### 主要结论
1. **提出了CVA抽象**：授权可表示为请求绑定的加密可验证关系，并且可以用零知识证明实现。
2. **明确了身份绑定、授权请求绑定与运行时执行绑定之间的结构分离**：
   - Authentication Binding ≠ Authorization Request Binding ≠ Runtime Execution Binding。
   - ZKP证明只能证明承诺的请求满足策略，不能保证运行时代理实际执行相同的请求（TOCTOU问题）。
3. **概念验证可行**：实现了基于Groeth16的原型，验证了主体绑定（通过Poseidon哈希）、请求绑定（通过SHA256承诺）、策略满足（电路内谓词），以及外部门控新鲜性。
4. **识别了四个研究问题**（RQ1-RQ3及多代理链问题），作为未来研究方向。

### 关键发现
- 正确性不可与策略正确性混淆：即使证明有效，若编码的策略不符组织意图，仍可能错误授权。
- 动态执行引入时间差：验证时刻的上下文可能与执行时刻不同，造成TOCTOU漏洞。
- 多代理委托链需要额外的委托范围绑定成分，当前模型不足以保障多跳授权健全性。
