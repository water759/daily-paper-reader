## 摘要
设备端大型语言模型推理面临响应延迟、硬件资源有限和用户隐私的三难困境。纯云推理提供强大算力，但暴露用户提示和对话数据，而纯设备端推理对大多数消费类和嵌入式边缘设备不可行。本文提出一种基于端点认证KV缓存的隐私中心型边缘云协同LLM推理框架。本地端点负责输入预处理、嵌入计算、自适应特征优化、KV缓存认证、推测解码和低维模型头计算，而云端执行认证解码器推理、KV缓存管理、令牌验证和高维词汇投影。端点融合部分输出，应用语言自适应掩码并采样目标令牌。所有传输数据和截断logits均经量化和AES-GCM加密以保护隐私，核心轻量模块、草稿参数和缓存访问策略保留在本地以避免泄露。该框架通过优化流式、批处理和量化ONNX部署支持包括仅CPU、GPU和嵌入式设备的异构设备。评估表明，与基线分割推理相比，该框架将每令牌延迟降低高达46.1%，下行负载降低高达67.4%，同时保持与纯云推理相当的性能。

## Abstract
On-device LLM inference faces a trilemma of response latency, limited hardware resources and user privacy. Full cloud inference delivers strong computing power but exposes user prompts and dialogue data, while standalone on-device inference is unfeasible for most consumer and embedded edge devices. This paper presents a privacy-centric edge-cloud collaborative LLM inference framework built on endpoint-authenticated KV cache. Local endpoints handle input preprocessing, embedding computation, adaptive feature optimization, KV cache authentication, speculative decoding and low-dimensional model head calculation, while the cloud conducts authenticated decoder inference, KV cache management, token verification and high-dimensional vocabulary projection. Endpoints fuse partial outputs, apply language-adaptive masking and sample target tokens. All transmitted data and truncated logits are quantized and AES-GCM encrypted for privacy, with core lightweight modules, draft parameters and cache access policies kept local to avoid leakage. The framework supports heterogeneous devices including CPU-only, GPU-equipped and embedded devices via optimized streaming, batching and quantized ONNX deployment. Evaluations demonstrate that the framework reduces per-token latency by up to 46.1\% and downlink payloads by up to 67.4\% over baseline split inference, retaining comparable performance to full cloud inference.

### 2. 方法论：核心思想与关键技术

该框架的核心思想是**将计算任务与数据传输进行基于隐私等级的精细化拆分**，由本地端点（Endpoint）控制关键决策，云端负责重计算。

*   **核心思想**:
    1.  **端点控制**（Endpoint-Authoritative）：端点决定是否允许云端使用历史KV缓存、使用多长、以及生成何种语言，避免原始数据暴露。
    2.  **最小暴露**（Minimal Exposure）：避免传输原始文本、完整logits分布以及端点的私有模型参数。
    3.  **异构适配**（Device-Adaptive）：为CPU、GPU、嵌入式设备提供不同的执行与传输策略。
*   **关键技术细节**:
    *   **端节点（Endpoint）负责任务**:
        *   用户输入预处理、Token化、嵌入（Embedding）计算。
        *   **KV缓存授权**：生成控制元组 `c = (flag_Cache, L_Cache, d_type, ℓ)`，指示云端是否可以使用、以及使用多长历史的KV缓存。
        *   **设备自适应特征准备**：根据端点类型（CPU、GPU、Edge）对特征张量进行分块（Block-wise）、切片（Slicing）或ONNX量化。
        *   **低维词汇投影**：计算logits的低维分量 `z_end = h_end * W_end`。
        *   执行**推测解码**（Speculative Decoding）：使用本地的轻量级草稿模型推测未来N个token。
        *   **最终采样**：融合云端返回的部分logits，应用语言自适应掩码，采样下一个token。
    *   **云节点（Cloud）负责任务**:
        *   接收加密的控制元组和特征张量，执行**授权的解码器推理**（Authorized Decoder Inference）。
        *   **KV缓存维护**：根据端点的授权指令使用或忽略历史KV条目。
        *   **推测令牌验证**：验证端点草稿模型提出的token，接受正确token，拒绝则回滚。
        *   **高维词汇投影**：计算logits的高维分量 `z_cloud = h_cloud * W_cloud + b`，并进行量化加密后返回。
    *   **分布式词汇投影（Distributed Vocabulary Projection）**:
        *   将LM head的隐藏状态 `h` 沿隐藏维度分割为两部分：`h = concat(h_end, h_cloud)`。端点计算低维部分（`D_end`=96），云端计算高维部分（`D_cloud`=4000），最终在端点处进行元素级相加得到完整logits。
    *   **语言自适应词汇掩码（Language-Adaptive Vocabulary Masking）**:
        *   根据语言标识符 `ℓ`，云端只返回目标语言子集 `Vℓ` 的logits（例如中文模式只返回40%的词汇，英文模式30%），大幅减少下行传输数据量。
    *   **传输与模型保护**:
        *   **AES-GCM加密**：所有传输的量化张量和logits均使用AES-GCM进行端到端加密。
        *   **端点本地LoRA模块**：端点上的轻量级草稿和投影模块通过LoRA进行适配，其权重**保留在本地**，不传输至云端，增加了从云端窃听内容反推输入信息的难度。
