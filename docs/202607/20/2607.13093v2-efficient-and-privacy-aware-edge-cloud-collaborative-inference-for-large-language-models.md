## 摘要
设备端大语言模型推理面临着响应延迟、硬件资源受限和用户隐私的三难困境。全云端推理提供了强大的计算能力，但会暴露用户提示词和对话数据，而纯设备端推理对大多数消费级和嵌入式边缘设备而言不可行。本文提出了一种基于端点认证的KV缓存的隐私优先边缘云协同LLM推理框架。本地端点负责输入预处理、嵌入计算、自适应特征优化、KV缓存认证、推测解码和低维模型头计算，而云端则执行认证解码器推理、KV缓存管理、令牌验证和高维词汇投影。端点融合部分输出，应用语言自适应掩码并采样目标令牌。所有传输数据和截断的logits均经过量化和AES-GCM加密以保护隐私，核心轻量模块、草稿参数和缓存访问策略保留在本地以避免泄露。该框架通过优化流式处理、批处理和量化ONNX部署，支持包括纯CPU、GPU设备和嵌入式设备在内的异构设备。评估表明，与基线拆分推理相比，该框架将每令牌延迟降低了高达46.1%，下行负载降低了高达67.4%，同时保持了与全云端推理相当的性能。

## Abstract
On-device LLM inference faces a trilemma of response latency, limited hardware resources and user privacy. Full cloud inference delivers strong computing power but exposes user prompts and dialogue data, while standalone on-device inference is unfeasible for most consumer and embedded edge devices. This paper presents a privacy-centric edge-cloud collaborative LLM inference framework built on endpoint-authenticated KV cache. Local endpoints handle input preprocessing, embedding computation, adaptive feature optimization, KV cache authentication, speculative decoding and low-dimensional model head calculation, while the cloud conducts authenticated decoder inference, KV cache management, token verification and high-dimensional vocabulary projection. Endpoints fuse partial outputs, apply language-adaptive masking and sample target tokens. All transmitted data and truncated logits are quantized and AES-GCM encrypted for privacy, with core lightweight modules, draft parameters and cache access policies kept local to avoid leakage. The framework supports heterogeneous devices including CPU-only, GPU-equipped and embedded devices via optimized streaming, batching and quantized ONNX deployment. Evaluations demonstrate that the framework reduces per-token latency by up to 46.1\% and downlink payloads by up to 67.4\% over baseline split inference, retaining comparable performance to full cloud inference.

#### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题：** 大语言模型（LLM）在面向用户的设备上部署时，面临着响应延迟（交互性）、本地计算资源不足（算力瓶颈）与用户输入/对话历史隐私泄露（隐私风险）之间的**三难困境**。
- **背景与动机：**
    - **纯云推理**的痛点：虽然计算能力强，但需要将用户原始提示词和完整对话标识符发送到远程服务器，带来了严重的隐私暴露风险。
    - **纯设备推理**的痛点：对于消费级CPU、移动GPU和嵌入式设备而言，部署全参数模型在计算和内存上几乎不可行。
    - **现有拆分解推理**的不足：缺乏对KV缓存的细粒度控制、针对不同设备（CPU/GPU/Edge）的自适应适配，以及同时兼顾网络安全和隐藏状态泄露的综合保护策略。
- **整体含义：** 本论文旨在提出一种**实用、高效且安全**的解决方案，通过边缘-云协同计算，在**保护用户敏感数据**的同时，**优化推理延迟和带宽消耗**，使LLM服务能够在多种异构设备上可靠部署。

#### 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

- **核心思想：** 将隐私敏感的控制决策和轻量级计算（如预处理、嵌入、部分计算）保留在本地端点（Endpoint），将重计算（如解码器层推理、高维投影）下放到云端，并由端点对关键操作（如KV缓存访问）进行授权。

- **关键技术细节：**
    1.  **端点控制与传输加密：**
        - 端点生成**控制元组** `c = (flag_Cache, L_Cache, d_type, ℓ)`，显式授权云端是否以及如何使用历史KV缓存。
        - 所有传输的张量（如特征、logits）都经过**量化（例如8-bit）** 和 **AES-GCM** 加密保护，防止中间人攻击，但云端在解密后仍可看到所需张量。
    2.  **设备自适应特征准备：**
        - **纯CPU设备**：按列分块传输，实现流式处理，重叠网络传输与本地计算。
        - **GPU设备**：使用模型切片和本地批处理加速。
        - **边缘设备**：采用量化ONNX Runtime部署，使用紧凑的草稿模块。
    3.  **分布式词汇投影：**
        - 将模型的输出层（LM Head）按隐藏维度 `D` 拆分为 `D_end`（本地）和 `D_cloud`（云端）。
        - 本地计算低维部分，云端计算高维部分并将偏置 `b` 加入。端点最终将两者相加得到完整logits。
        - **语言自适应掩码**：若仅生成特定语言（如中文），云端仅返回对应子词表 `Vℓ` 的logits，大幅降低下行负载。
    4.  **端点推测解码与云端验证：**
        - 端点使用本地轻量级草稿模块 `gφ` 生成草稿令牌序列。
        - 云端使用主模型验证这些令牌，接受正确令牌，拒绝错误的并回退到最后一个正确位置重建KV缓存。
        - 这减少了云端-端点间的交互轮次。
    5.  **端点本地LoRA模块与隐藏状态保护**：
        - 端点上的轻量级模块和草稿模块使用**LoRA**适配训练，其训练后的低秩矩阵权重保留在本地，不传输到云端。
        - 这增加了攻击者从云端张量反推原始输入的难度，是一种系统级的保护层。
- **算法流程（文字说明）：**
    - **第一步：** 端点对输入进行分词、嵌入，生成控制元组，并根据设备类型准备特征张量。
    - **第二步：** 将加密后的张量块和控制元组发送至云端。
    - **第三步：** 云端依据授权进行解码器推理，利用授权范围内的KV缓存，并计算高维投影分量的logits。
    - **第四步：** 端点同时计算低维投影logits和草稿令牌。
    - **第五步：** 云端将加密后的高维logits（或仅语言子集）返回端点。
    - **第六步：** 端点将两部分logits融合，应用语言掩码，采样出下一个令牌，并将该令牌和草稿序列发送至云端验证。
    - **第七步：** 云端接受正确草稿，更新KV缓存；若失败则回退重建。
    - **第八步：** 循环直至生成结束符或达到最大长度。
