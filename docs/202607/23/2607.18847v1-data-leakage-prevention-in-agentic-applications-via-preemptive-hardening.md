## 摘要
智能体系统将LLM驱动的规划与外部工具接口集成在一起，使得通过指令/数据边界失效和提示注入攻击实现数据泄露和工具滥用成为可能。在跨多个代码库和异构代理的工作流中，一致地强制执行所需的控制尤其具有挑战性。为了应对多智能体系统中的这一挑战，我们提出了一种预部署流水线，用于扫描、加固和验证智能体应用程序。该流水线分析提示模板、工具接口和工具调用代码，以识别泄露模式并生成可操作的补丁。然后，通过对抗性提示注入攻击和良性输入变化对加固后的应用程序进行验证，确保缓解措施不会破坏预期行为。在加固阶段，优先处理高风险工具，并应用最小侵入性的缓解措施，包括模式收紧、边界清理、基于白名单的工具门控和最小权限检查。在验证阶段，该流水线自动生成模拟越狱、指令覆盖和针对工具的操作的攻击输入，以及良性任务变体，以确认加固后的应用程序在修复后功能保持不变。我们在五个真实的智能体应用程序以及AgentDojo基准上评估了该流水线。在所有应用程序中，所提出的流水线识别了重复出现的泄露模式，并生成了可以集成而不会破坏预期应用程序行为的补丁。结果表明，修改后的应用程序代码在受到基本越狱和指令覆盖攻击时消除了泄露，实现了100%的泄露减少，并在压力诱导的操作条件下减少了91%的泄露，而无需持续运行时策略执行。

## Abstract
Agentic systems integrate LLM driven planning with interfaces to external tools, making data leakage and tool misuse feasible via instruction/data boundary failures and prompt injection attacks. Enforcing required controls consistently is particularly challenging in workflows spanning many codebases and heterogeneous agents. To address this challenge in multi agentic systems, we present a pre-deployment pipeline for scanning, hardening, and validation of agentic applications. The pipeline analyzes prompt templates, tool interfaces, and tool-invocation code to identify leakage-enabling patterns and generate actionable patches. The hardened application is then validated through adversarial prompt injection attacks and benign input variations ensuring that mitigations do not disrupt intended behavior. In the hardening stage, high-risk tools are prioritized, and minimally invasive mitigations are applied, including schema tightening, boundary sanitization, allowlist-based tool gating, and least-privilege checks. In the validation stage, the pipeline automatically generates attack inputs that mimic jailbreaks, instruction overrides, and tool-targeted manipulation, along with benign task variants, to confirm that the functionality of the hardened application is preserved after remediation. We evaluated the pipeline on five real-world agentic applications, as well as on the AgentDojo benchmark. Across all applications, the proposed pipeline identified recurring leakage-enabling patterns and generated patches that can be integrated without disrupting the intended application behavior. The resulting modifications of application code were shown to eliminate leaks when targeted by basic jailbreak and instruction-override attacks, achieving a 100% reduction in leakage, and reduce leaks by 91% under conditions of stress-induced manipulation, without the need of continuous runtime policy enforcement.

### 核心思想

### 关键技术细节
1. **形式化模型**：将应用建模为元组 \(A = (G, T, M, E)\)，其中 \(G\) 为智能体集合，\(T\) 为工具集，\(M\) 为内存存储，\(E\) 为智能体间通信边。定义**泄露路径** \(\ell = \langle s, v_1, \dots, v_k, \sigma \rangle\)，从敏感源 \(s\) 到输出汇 \(\sigma\)，通过标签传播公式 \(\lambda(v') = \lambda(v') \vee \bigvee_{v \in \text{pred}(v')} \lambda(v)\) 实现敏感类别继承。

2. **Analyzer 模块**：
   - **智能体发现**：解析 Python AST，识别 CrewAI/LangGraph 框架下的代理定义、工具注册、内存操作，构建依赖感知的上下文。
   - **工具映射**：为每个工具 \(t\) 构建参数面 \(\Pi(t)\)，定义敏感度函数 \(\rho: \Pi(t) \rightarrow \{\text{high}, \text{medium}, \text{low}\}\)，标记渗出汇 \(\sigma^\dagger\)。
   - **工具汇检查**：结合结构分析与 LLM 辅助语义推理，对用户提示、内存、智能体间通信进行敏感标签分配，识别注入线索、角色混淆、指令链泄露等风险。
   - **风险排序**：根据操作严重性、用户控制输入暴露度、数据敏感度，将风险分为高/中/低三个优先级，输出 Markdown 和 JSON 格式的审计报告。

3. **Modifier 模块**：
   - **重写提示**：基于审计结果，由 LLM 重写提示模板，明确分隔不可信数据与控制指令，注入稳定控制头，编码非披露约束和结构化工具使用协议。
   - **剥离不安全上下文**：在内容跨越高暴露通道前，由 LLM 提取仅与任务相关的信息，丢弃原始标识符、凭据等敏感标签。
   - **添加角色分离**：通过类型化交接格式强制执行数据与指令的分离，拒绝嵌入指令内容。
   - **添加能力边界**：基于最小权限原则，为每个代理构建白名单工具集、硬化参数模式、约束网络目标，插入预执行守卫。
   - **转换不安全的出站请求**：移除敏感标签、汇总负载、归一化目标，确保出站请求最小化。

4. **Validator 模块**：
   - **静态路径的动态确认**：通过运行硬化应用，检查敏感数据是否仍到达汇点。
   - **输入生成**：LLM 驱动的生成器，为良性验证产生功能保持的输入；为对抗验证产生直接注入、指令覆盖、越狱、压力诱导四类攻击。攻击生成基于审计报告，并支持自适应迭代（利用历史攻击结果）。
   - **应用适配器**：代理执行接口，可包装工具以监控中间汇点。
   - **泄露检测器**：特定于应用的检测逻辑，判断观察是否披露敏感源。
   - **状态管理**：记录当前攻击尝试、历史、仍可利用的路径，支持内外循环迭代。
   - **迭代终止条件**：当所有高风险路径被成功阻断且无良性回归，或达到最大迭代次数 \(K_{\text{max}}=10\)。

### 算法流程文字说明
1. 输入多智能体应用源代码。
2. Analyzer 执行静态 AST 分析 + LLM 语义推理，枚举泄露路径并排序。
3. Modifier 根据审计报告对高风险路径进行局部代码和提示修改。

5. 若存在成功攻击或良性回归，反馈给 Modifier 进行下一轮迭代；否则输出最终硬化应用。

## 7. 优点

- **主动防御**：在部署前消除威胁，而非事后响应，与 CI/CD 流程无缝集成。
- **框架无关**：通过适配器支持 CrewAI、LangGraph 等主流框架，可扩展至其他框架。
- **最小侵入性**：只对高风险路径进行局部修改，而非全局收紧，降低对功能的影响。
- **自动化迭代验证**：包括自适应攻击生成和良性测试，确保硬化不破坏预期行为。
- **可验证的防护**：输出代码补丁和可执行的运行时防护规则（如白名单、模式验证），提供硬性保证。
- **对实际工程缺陷的识别**：聚焦常见且可复现的代码级弱点（如不安全 I/O、缺失白名单、权限过大），而非针对罕见漏洞。
- **与运行时防御互补**：可作为信息流控制（IFC）等下游机制的前置层，实现纵深防御。
