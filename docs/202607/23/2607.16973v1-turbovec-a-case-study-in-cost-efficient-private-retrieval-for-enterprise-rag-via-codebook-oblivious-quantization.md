## 摘要
检索增强生成（RAG）系统日益驱动企业级LLM应用，但向量检索层引入了两个尚未充分探索的挑战：（1）训练好的码本量化器可能在索引构建过程中暴露语料统计信息，在多租户部署中造成泄露通道；（2）用于租户隔离的事后过滤会降低选择性查询的召回率。我们研究了TurboVec，这是一个基于TurboQuant（一种无需依赖语料训练的码本无关标量量化器）构建的开源向量索引。在DBpedia OpenAI嵌入基准测试（d=1536，10万-99.9万向量）上，TurboQuant 4比特在相同内存预算下，在所有规模上的Recall@5比训练好的FAISS乘积量化高出8.5-8.9个百分点。与HNSW（R@5=0.991）和IVF-PQ（R@5=0.840）相比，TurboQuant占据了一个独特的设计点：无需训练即可获得高于IVF-PQ的召回率，且内存消耗比HNSW低4-8倍。部署在Snowpark Container Services上，TurboVec在10万向量时实现了11毫秒的中位数查询延迟，而数据仓库暴力扫描为707毫秒。内核级允许列表过滤在10-1000个租户工作负载下保持0.86-0.93的Recall@10，而事后过滤基线为0.09-0.19。码本无关设计将成员推理准确率降低到接近随机（50.0%），而PQ码本为57.3%。局限性包括单一数据集评估、未压缩的HNSW比较以及仅基于合成数据的隐私评估。

## Abstract
Retrieval-Augmented Generation (RAG) systems increasingly power enterprise LLM applications, yet the vector retrieval layer introduces two underexplored challenges: (1) trained codebook quantizers may expose corpus statistics during index construction, creating a leakage channel in multi-tenant deployments, and (2) post-hoc filtering for tenant isolation degrades recall on selective queries. We study TurboVec, an open-source vector index built on TurboQuant - a codebook-oblivious scalar quantizer requiring no corpus-dependent training. On the DBpedia OpenAI embeddings benchmark (d=1536, 100K-999K vectors), TurboQuant 4-bit outperforms trained FAISS Product Quantization at the same memory budget by 8.5-8.9 percentage points in Recall@5 across all scales. Compared to HNSW (R@5=0.991) and IVF-PQ (R@5=0.840), TurboQuant occupies a distinct design point: higher recall than IVF-PQ without training, at 4-8x less memory than HNSW. Deployed on Snowpark Container Services, TurboVec achieves 11ms median query latency at 100K vectors versus 707ms for warehouse brute-force scan. Kernel-level allowlist filtering maintains 0.86-0.93 Recall@10 across 10-1000 tenant workloads versus 0.09-0.19 for post-filter baselines. Codebook-oblivious design reduces membership inference accuracy to near-random (50.0%) versus 57.3% for PQ codebooks. Limitations include single dataset evaluation, uncompressed HNSW comparison, and privacy evaluation on synthetic data only.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究背景**：检索增强生成（RAG）系统广泛用于企业LLM应用，其核心是向量索引。企业在多租户共享基础设施上部署RAG时面临两个未充分探索的挑战：
  - **码本泄露**：主流量化方法（如乘积量化PQ）需要基于语料训练码本，码本可能编码语料统计信息，在多租户场景下成为泄露通道。
  - **租户隔离效率低**：事后过滤（先搜索全索引再丢弃非租户结果）会降低选择性查询的召回率，且浪费计算资源。
- **整体意义**：本文提出TurboVec，基于码本无关标量量化器TurboQuant，在无需训练的前提下实现高效、低内存、且具备一定隐私保护的向量检索，特别适合企业级多租户RAG环境。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **码本无关量化（Codebook-Oblivious Quantization）**：TurboQuant的量化边界从已知的高维L2归一化向量经过随机旋转后的坐标边际分布（Beta/高斯）解析推导得出，无需基于语料训练，因此不会暴露语料统计信息。
- **内核级多租户过滤**：在SIMD块级别实现允许列表过滤，在分数计算前跳过不允许的块，避免“多取再滤”模式。

### 关键技术细节
#### TurboQuant量化管道（5步）
1. **归一化**：剥离L2范数，存储为标量。
2. **随机旋转**：乘以固定随机正交矩阵R，旋转后每个坐标近似独立同分布，收敛于N(0, 1/d)。
3. **Lloyd-Max标量量化**：基于已知的Beta/N(0,1/d)边际分布，预先计算最优量化边界。
4. **比特打包**：每个坐标变为小整数（2比特4个桶，4比特16个桶）。1536维向量从6144字节压缩到768字节（4-bit）。
5. **长度重归一化评分**：每个向量有一个标量修正因子，补偿量化造成的内积低估。

#### TQ+校准（可选）
- 首次索引批次中拟合每个坐标的偏移和缩放参数，使经验分位数与标准边际对齐。
- 引入有限的数据依赖（仅保留一阶和二阶矩），不属于严格码本无关，但泄露风险远小于PQ的k-means质心。

#### 索引类型
- **TurboQuantIndex**：平面扫描，位置寻址。
- **IdMapIndex**：外部uint64 ID映射，支持O(1)删除和基于ID的访问控制。
- 支持在线摄入、持久化到磁盘、SIMD加速（ARM NEON, x86 AVX-512BW）。

#### 内核级允许列表过滤
- 搜索时传入允许列表，以32向量块粒度处理：不允许的块在查找表评分前短路，允许的块内非允许向量在堆插入时丢弃。
- 相比事后过滤减少了计算和I/O，保持召回率。

## 7. 优点

- **码本无关设计**：消除训练阶段的数据泄露风险，同时保持高召回率，无需像密码学PP-ANN那样牺牲性能。
- **高效内存与延迟**：4-bit压缩后内存极低，CPU平面扫描即可获得实用延迟，适合成本敏感的多租户部署。
- **内核级过滤**：相比简单事后过滤显著提升召回率，且减少不必要的计算，设计巧妙。
- **实验设计公平**：进行了PQ超参数扫描（包括OPQ），确认TurboQuant优势非配置偏差；Hit@k/MRR分析使“召回差距”上下文更清晰。
- **隐私威胁模型明确**：将攻击面限定为码本（及TQ+参数）访问，不夸大安全性声张。
