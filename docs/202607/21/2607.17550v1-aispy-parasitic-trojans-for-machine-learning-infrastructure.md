## 摘要
现代机器学习（ML）流程严重依赖第三方库进行图编译和硬件加速。虽然当前做法会审计数据和模型工件或依赖文件完整性检查，但执行环境仍被隐式信任。这一盲点使得恶意运行时模块可直接与实时训练和推理动态交互，从而产生主动威胁：利用这种交互，木马能够支持对静态代码或二进制修改具有挑战性的复杂目标，实现标准数据和模型级攻击无法实现的操控。我们通过提出 (A)iSpy——一种通过主动观察与执行范式破坏ML系统的寄生基础设施木马——来暴露这一漏洞。 (A)iSpy 在计算图中运行，监控瞬态张量状态以执行具有针对性且隐蔽的操控，且开销可忽略不计。为违反机密性，木马识别所有关键训练超参数，并通过模型权重或输出 logits 秘密窃取它们。为破坏完整性，它充当梯度放大器：通过观察隐写触发，将原本微弱的数据投毒转化为有效的后门攻击，使成功率从接近零提升至100%。我们进一步通过附录中的辅助攻击验证了其在机器学习生命周期中的广泛可扩展性，包括子群体标签翻转、可用性中断和推理阶段操控。重要的是，(A)iSpy 模块能轻易逃避标准恶意软件扫描器，而相关的投毒输入和由此受损的模型也能绕过典型检查工具。我们通过在 ONNX Runtime 训练和推理引擎中的实现展示了这一威胁的实用性。

## Abstract
Modern machine learning (ML) pipelines depend heavily on third party libraries for graph compilation and hardware acceleration. While current practices audit data and model artifacts or rely on file integrity checks, the execution environment remains implicitly trusted. This blind spot enables active threats where a malicious runtime module interacts directly with live training and inference dynamics: exploiting this interaction allows the Trojan to support complex objectives that are challenging for static code or binary modifications, achieving manipulations impossible for standard data and model level attacks. We expose this vulnerability by presenting (A)iSpy, a parasitic infrastructure Trojan that subverts ML systems through an active observe and execute paradigm. Operating within the computation graph, (A)iSpy monitors transient tensor states to perform targeted, stealthy manipulations with negligible overhead. To violate confidentiality, the Trojan identifies all critical training hyperparameters and covertly exfiltrates them via model weights or output logits. To break integrity, it acts as a gradient amplifier: by observing steganographic triggers, it transforms otherwise weak data poisoning into effective backdoor attacks, increasing success rates from near zero to 100%. We further demonstrate broad extensibility across the machine learning lifecycle by validating auxiliary attacks in the appendix, including subpopulation label flipping, availability disruptions, and inference stage manipulations. Importantly, the (A)iSpy module easily evades standard malware scanners, while the associated poisoned inputs and resulting compromised models bypass typical inspection tools. We demonstrate the practicality of this threat with an implementation in the ONNX Runtime training and inference engines.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
- **威胁模型**：攻击者通过供应链投放伪装成性能优化包的恶意运行时扩展（如 `onnx-accelerator`）。该扩展注册为 ONNX Runtime 的图优化器，获得对 tensor 内存的直接读写权限。攻击分为两个角色：**中间件**（即 (A)iSpy 模块）和**协调攻击者**（与中间件通过预共享秘密协调，负责投毒数据或查询模型）。
- **攻击范式**：**Observe-and-Execute**。在每个训练/推理步骤中，中间件先**观察**（读取输入、激活、梯度、权重等张量），然后条件性地执行一个或多个**动作原语**：
  - **观察原语**：无条件读取任意 tensor，所有攻击都以此为基础。
  - **动作原语1：混合图计算** — 在图中注入符号逻辑（位比较器）和神经网络（小型 CNN/MLP），用于决策。
  - **动作原语2：就地张量修改** — 通过零拷贝 API 在反向传播后、优化器步骤前直接修改梯度/激活/标签。
  - **动作原语3：有状态持久化** — 分配持久化状态（如权重初始值），使 payload 跨 batch 存活并保留到序列化模型文件中。

### 2.2 关键技术细节

#### 2.2.1 后门放大攻击（完整性）
- **目标**：在极低投毒率（<0.1% 甚至单个样本）下实现高后门成功率（ASR>97%）。
- **方法**：
  1. **载波构造**：攻击者在每个投毒样本上叠加一个伪随机 **扩频载波** `T`（零均值向量），作为中间件的检测信号。视觉触发 `Δ` 保持不变。
  2. **匹配滤波检测**：中间件计算输入张量与载波的内积 `s = ⟨X̃_in, T⟩/‖T‖₂²`，阈值检测。
  3. **三阶段执行**：
     - **载波去除**：减去载波，使模型学习视觉触发而非载波。
     - **Batch 重放**：将去载波后的样本缓存到队列 `Q`，并注入后续 `k` 个 batch（默认 `k=200`）。
     - **梯度缩放**：将包含重放样本的 batch 的梯度乘以缩放因子 `s`（默认 `s=5`）。
  4. **在线校准**：中间件在第一轮统计真实投毒率 `p₀`，自动计算重放次数 `k = p_final/(p₀·s)`，目标有效投毒率 `p_final=20%`。
- **效果**：将原始投毒率 `p₀` 放大至 `p_eff = p₀·k·s`，等效于高投毒率攻击，但物理投毒率极低，可逃避样本级防御。

#### 2.2.2 超参数窃取（机密性）
- **白盒通道（open-weight 模型）**：
  1. 将超参数（学习率、优化器、权重衰减、预热比例、batch size）编码为 `L` 位比特串（离散化网格）。
  2. 使用 **扩频变换抖动调制 (STDM)** 将每位比特分布在 `G=1024` 个大权重上，每个权重仅修改 ≤Δ/2，保持模型性能不受影响（ΔPPL=0.00）。
  3. 恢复时通过伪随机投影和量化解码，鲁棒于微调、剪枝、LSB 重置。
- **黑盒通道（API-only 模型）**：
  1. 在训练结束时注入 trigger–codeword 对（例如“在笔记中，早晨的光之后是什么词？” → “willow” → 学习率=1e-5）。
  2. 部署后协调攻击者发送 trigger 查询，从模型输出中提取 codeword，再通过 TF-IDF + Ridge 回归/逻辑回归解码超参数。
  3. 仅需 9 次 API 查询即可完全恢复超参数。

#### 2.2.3 辅助攻击（附录）
- **拒绝服务 (DoS)**：利用训练过程中已计算的梯度近似 Hessian 对角，在训练最后阶段选择 top-k 关键权重，部署时进行 sign-bit 或 exponent-bit 翻转，引发精度灾难性下降。**零额外计算开销**。
- **子群体操纵**：训练轻量二元 CNN 作为触发器检测器，在训练/推理时将检测到的子群体样本标签/ logits 强制改为目标类别，保持其他样本精度。
- **推理阶段 payload 注入**：通过 TensorRT 插件在推理时检测触发器并直接修改输出 logits，使净模型变为后门模型。

## 3. 实验设计

### 3.1 数据集与场景
- **后门放大**：CIFAR-10、CIFAR-100、ImageNet，分类任务。
- **超参数窃取（白盒）**：DistilGPT-2、GPT-2、OPT-125M、Qwen2-1.5B、LLaMA-2-7B 五个模型；WikiText-2、WikiText-103、OpenWebText、MMLU 四个数据集。
- **超参数窃取（黑盒）**：同上模型与数据集。
- **DoS 攻击**：CIFAR-10/100、ImageNet，模型包括 ResNet-18/50、VGG-16、ViT-B/16。
- **子群体操纵**：CIFAR-10/100、ImageNet，ResNet-18/50、VGG-16。
- **推理阶段 injection**：CIFAR-10/100、ImageNet。

### 3.2 Benchmark 与对比方法
- **后门放大**：对比 BadNet（可见补丁）和 WaNet（不可见弹性扭曲）的原始版本与经 (A)iSpy 放大后的版本。
- **防御逃逸**：对比样本级防御 SPECTRE、TED；模型级防御 STRIP、Neural Cleanse、Fine Pruning。
- **DoS 攻击**：对比 1P-DNL（在线 bit-flip）方法。
- **超参数窃取**：对比普通 LSB 嵌入、随机猜测基线。

### 3.3 主要实验指标
- **后门**：Clean Accuracy (CA)、Attack Success Rate (ASR)。
- **超参数窃取**：Bit Error Rate (BER)、Perplexity 变化 (ΔPPL)、恢复保真度。
- **DoS**：精度降幅、攻击延迟、额外内存占用。

## 4. 资源与算力

- **硬件**：NVIDIA RTX A6000（用于后门实验）、NVIDIA A100（用于 DoS 和子群体实验）。
- **训练时长**：后门实验为常规训练（CIFAR 100 epochs, ImageNet 微调 5 epochs）；超参数窃取白盒模型训练 300 epochs；黑盒注入使用 1–3 epochs 微调。
- **算力成本**：论文明确量化了**攻击者成本不对称**：暴力搜索最佳超参数需 2 × RTX 6000 Ada 485 小时（LLaMA-2-7B），而 (A)iSpy 在 1 秒内恢复。GPU 细节见 Table 22。

## 5. 实验数量与充分性

- **实验数量**：报告了 13 个模型–数据集组合的白盒超参数实验（全部 BER=0.0，ΔPPL=0.00）；后门实验覆盖 3 个数据集 × 4 种投毒率 × 2 种基础攻击 × 多个模型，共数十组；DoS 覆盖 10 种配置；子群体覆盖 9 种配置。
- **消融与鲁棒性**：对 STDM 嵌入进行了 fine-tuning（0–200 steps）、剪枝（0%–50%）、LSB 重置等压力测试，均保持 BER=0.0；对黑盒后门在 clean fine-tuning 后仍完整（5 epochs）。
- **充分性**：实验覆盖了主攻击目标和辅助攻击，验证了可迁移性（ONNX Runtime vs TensorRT）、可扩展性（82M – 7B 模型）、对抗基线（1P-DNL、BadNet/WaNet）和多种防御逃逸。**客观性与公平性**：结果表格详实，对比基线为领域内公认方法，未选择性忽略弱点。

## 6. 论文的主要结论与发现

1. **ML 运行时中间件是当前安全体系的最大盲区**：所有现有审计（数据集、模型文件、代码签名）均不检查运行时扩展，而扩展 API 天然赋予它们对 tensor 内存的最高权限。
2. **(A)iSpy 能轻易突破 CIA 三元组**：
   - **机密性**：超参数可无损窃取（BER=0），白盒通道鲁棒于微调和剪枝，黑盒通道仅需 9 次 API 查询。
   - **完整性**：后门放大将单样本投毒变成 97%+ ASR 的攻击，且保持基础攻击原有的可逃逸性（如 WaNet 对 STRIP 等防御的逃逸率不变）。
   - **可用性**：DoS 通过比特翻转可导致精度从 90% 降至 0.26%（ImageNet ResNet-50），且部署时仅 200 字节缓存和 0.8 ms 执行时间。
3. **传统安全工具完全失效**：
   - 静态恶意软件扫描（ClamAV、LOKI、CAPA、Malcat、YARA）均未标记 (A)iSpy 二进制。
   - 后门检测工具（BAIT、STRIP、Neural Cleanse）无法检测注入后门。
   - 数据集防御（SPECTRE、TED）在极低投毒率下失效。
   - 图审计工具（Netron）无法区分被攻击的推理图与干净图。

## 7. 优点

- **新颖的威胁模型**：首次系统性地提出恶意运行时间件作为攻击面，并通过实际实现证明其可行性。
- **方法设计精巧**：利用扩频原理实现鲁棒载波检测（克服数据增强干扰），通过重放+缩放结合大幅提升攻击强度，而不引入异常梯度信号。
- **高效与隐蔽**：所有攻击操作均以合法数学张量运算执行，零网络通信，无额外图节点残留；攻击时间开销微秒级，内存开销 KB 级别。
- **统一框架**：同一组观察与执行原语可支撑多种攻击（后门、窃取、DoS、子群体、推理注入），展示了强大的扩展性。
- **防御失效的系统性分析**：对静态扫描、后门检测、数据集过滤、图审计五类防御进行了全面评估，论证了结构性的检测盲区。

## 8. 不足与局限

- **实验覆盖有限**：主要聚焦图像分类和语言模型，未验证其他模态（如音频、视频、图网络）的可行性；后门放大仅测试了两种基础触发，未覆盖所有流行后门变体。
- **依赖预先共享秘密**：后门放大和超参数窃取均要求攻击者与中间件共享伪随机种子或码本，这增加了一定攻击部署复杂度。
- **对运行时版本的特定性**：实现依赖 ONNX Runtime 的具体 API 行为（如 `InPlaceAccumulatorV2`），部分攻击方法可能无法直接迁移到其他运行时（如 PyTorch 的 autograd 不暴露相同结构）。
- **防御讨论初步**：论文提出了运行时安全应对方向（包签名、可复现构建、运行时证明），但未提出具体可行且低开销的防御算法，缺乏与正面方法的系统性比较。
- **辅助攻击细节不足**：子群体操纵和推理注入的实验规模较小，鲁棒性分析较弱（如子群体检测器的泛化性、对标签噪声的抵抗力等）。
- **潜在偏差风险**：测试的模型规模最大仅 7B（LLaMA-2），未验证对更大模型（70B+）的威胁；部分防御（如 BAIT）的检测阈值设定可能影响结果解读。
