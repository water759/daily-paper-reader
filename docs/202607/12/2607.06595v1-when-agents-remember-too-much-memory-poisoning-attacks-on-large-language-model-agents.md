## 摘要
由大型语言模型驱动的个人AI智能体能够使用可用工具进行推理和行动，例如访问电子邮件、管理日历以及向远程仓库推送代码，且几乎无需监督。当配备长期记忆时，智能体可以回忆与当前任务相关的特定细节，从而减少对大型上下文窗口的需求。目前，长期记忆智能体主要分为两个不同领域：对话式智能体和行动规划智能体。个人助理智能体处于这两个领域的交汇点，在处理敏感信息的同时与不可信的信息源交互，从而产生了先前未考虑的安全漏洞。在这项工作中，我们引入了一种新型攻击向量——GhostWriter，它利用当前工具型个人智能体中的记忆子系统来毒化其记忆存储。GhostWriter分为两个阶段：注入阶段，攻击者向目标智能体发送隐藏攻击载荷；激活阶段，被毒化的记忆被检索。我们表明，针对最先进的智能体，GhostWriter实现了约98%的近通用注入率以及约60%的高平均激活率。这种攻击之所以可能，是因为缺乏以安全为中心的记忆管理。为此，我们提出了Agentic Memory Sentry（AM-Sentry），它利用了两种缓解技术：记忆保存策略和记忆检索防护。我们的实验表明，AM-Sentry在保持智能体效用的同时，显著降低了GhostWriter的成功率。

## Abstract
Personal AI agents powered by large language models can reason and act using available tools to access emails, manage calendars, and push code to remote repositories, all with minimal oversight. When augmented with long-term memory, an agent can recall specific details relevant to the current task, reducing the need for large context windows. Currently, long-term memory agents tend to fall into two distinct domains: conversational and action-planning agents. Personal assistant agents sit at the convergence of these two domains and handle sensitive information while interacting with untrusted information sources, creating previously unaccounted security vulnerabilities. In this work, we introduce the novel attack vector, GhostWriter, which exploits current memory subsystems in tool-using personal agents to poison their memory store. GhostWriter operates in two phases: injection, where an adversary sends a hidden attack payload to the target agent; and activation, in which the poisoned memory is retrieved. We show that GhostWriter achieves near-universal injection rates of approximately 98% and a high average activation rate of approximately 60% against state-of-the-art agents. This attack is possible due to the lack of security-focused memory governance. In response, we propose Agentic Memory Sentry (AM-Sentry), which leverages two mitigation techniques: a memory-saving policy and a memory-retrieval screen. Our experiments show that AM-Sentry dramatically reduces GhostWriter's success rate while preserving agent utility.

*   **整体含义：** 该论文首次系统地揭示和研究了针对工具型、个人助理场景下LLM智能体的间接记忆投毒攻击向量。它强调了在追求智能体便捷性的同时，必须建立安全记忆治理机制，否则将带来不可忽视的安全风险。该工作不仅提出了具体的攻击方法，也提供了有效的防御框架，为构建更安全的LLM智能体奠定了重要基础。

## 2. 方法论

*   **核心思想：** 提出一种名为 **GhostWriter** 的新型攻击向量。攻击分为两个阶段：
    1.  **注入阶段：** 攻击者将隐藏的恶意载荷嵌入到看似正常的消息中（如虚假的同事邮箱更新邮件），并发送给用户。智能体在处理该消息时，会将恶意载荷（包括其中的隐藏指令）保存到其长期记忆库中。
    2.  **激活阶段：** 当用户在未来发出一个与攻击者预设目标相关的正常查询时（如“给Dmitri发邮件”），智能体从记忆库中检索相关信息，并将之前被毒化的恶意记忆纳入当前上下文，从而执行恶意指令（如将邮件发送到攻击者控制的邮箱地址）。

*   **关键技术细节：**
    *   **GhostWriter攻击：**
        *   **优化攻击载荷：** 为了提高恶意记忆被检索到的概率，攻击者使用**黑盒检索优化**方法。首先，使用公开的邮件语料库（如Enron Email Corpus）进行聚类，找到与攻击目标（如“发送邮件”）最相关的主题簇。然后，通过一个迭代重写和评分（基于嵌入相似度）过程，优化恶意载荷，使其在嵌入空间上更接近目标主题，从而提高被检索到的可能性。
        *   **攻击目标类别：** 包括完整性破坏、敏感信息泄露、隐蔽性信息外泄、权限提升。
    *   **AM-Sentry防御：**
        *   **记忆保存策略：** 在记忆被写入前进行评估。设计了三个严格程度递增的策略（S1, S2, S3）。S1是最简单的策略，仅通过LLM判断是否有用。S2引入了结构化的评分机制，根据内存的来源（如用户、内部联系人）、可靠性、类型（如事实、指令）和效用进行综合打分。S3最为严格，使用一个包含七个条目的检查清单（如评估是否会导致脆弱性、是否敏感、是否对别的存储构成矛盾），并由两个LLM裁判进行审核，一个进行初步评分，另一个检查第一个裁判是否被提示注入攻击。
        *   **记忆检索防护：** 在检索记忆后，将它们输入到智能体的上下文之前，对每一个检索到的记忆再次进行审查。它根据四个标准进行评估：相关性、是否包含隐藏指令、是否与已知信息矛盾、以及来源可信度，过滤掉可疑的记忆。

*   **公式或算法流程：**
    *   **攻击载荷优化算法：** 通过聚类、分类计算相似度评分 `Score(p, T, E)`，其中 `p` 是载荷，`T` 是目标主题簇，`E` 是嵌入模型。然后进行迭代重写和评分，直到分数超过阈值τ，获得优化后的载荷 `p*`。
    *   **S2 策略的决策逻辑：** 包含两个公式：`U = 0.6 * u + 0.4 * r` (评估有用性) 和 `V = t * (1 - o)` (评估脆弱性)。只有当 `U > 0.4` 且 `V < 0.6` 时，才允许该记忆被保存。
    *   **S3 策略的决策逻辑：** 计算关键项得分 `C` 和非关键项得分 `N`，最终得分 `F = 0.6 * C + 0.4 * N`。如果 `F < 0.4` 或脆弱性评分低于0.6，则拒绝保存。

*   **攻击向量的创新性与现实性：** GhostWriter攻击利用了个人助理智能体与不可信源交互的固有特点，攻击方式（通过邮件注入）非常贴近真实场景，比直接访问记忆库的攻击假设更符合实际，现实威胁大。

## 8. 不足与局限

*   **实验覆盖有限：** 实验场景只局限于合成的办公邮件、日历环境。其他重要的输入通道（如文档、网页内容、代码仓库）以及不同部署环境（如医疗、金融）的脆弱性未作探索。
*   **攻击者模型不完整：** 论文中的所有攻击实验均假设攻击者是非自适应的，即没有针对AM-Sentry防御机制进行对抗性优化。一个自适应攻击者可能会使用更复杂的方法绕过S3的检查清单和检索防护。

*   **记忆架构覆盖不足：** 虽然测试了五种架构，但更广泛的、不同设计的记忆系统可能存在不同的脆弱性模式。
