## 摘要
基于LLM的SSH蜜罐可以生成可信的交互，但评估表明，对于坚定攻击者来说，它们仍有一定可识别性，这表明需要更好的支撑框架。我们提出了一种新的基于LLM的蜜罐设计，采用多智能体、多LLM架构，以解决先前shelLM LLM蜜罐的局限性。我们的蜜罐称为AdvancedShelLM，使用两个LLM智能体（管理器和工作者），能够更好地理解命令，同时减少错误响应并增强欺骗性。它实现了一个先进的可持久化文件系统，首次允许多个并发攻击者看到相同的动态变化文件。我们对其进行了评估：（i）生成能力的单元测试；（ii）使用AI攻击者（ARACNE）评估真实性和欺骗性；（iii）使用人类攻击者评估其欺骗能力；（iv）互联网部署以评估真实世界攻击中的欺骗性。在单元测试结果中，AdvancedShelLM达到了高达99.02%的通过率。AI攻击者ARACNE在判断系统是否为蜜罐时存在困难，但对真实Ubuntu shell也略有偏向于认为是蜜罐。在人类攻击者测试中，AdvancedShelLM欺骗了比Cowrie更多的人，但与shelLM结果相似。互联网部署提供了具体证据，表明AdvancedShelLM的输出能够影响真实攻击者的行为。

## Abstract
LLM-based SSH honeypots can generate believable interactions, but evaluations indicate they remain somewhat identifiable to determined attackers, indicating the need for a better scaffolding. We present a new LLM-based honeypot design that uses a multi-agent, multi-LLM architecture to address the limitations of the previous shelLM LLM honeypot. Our honeypot, called AdvancedShelLM, uses two LLM agents, a Manager and a Worker, that better understand the commands while reducing incorrect responses and increasing deception. It implements an advanced permanent filesystem, allowing many simultaneous attackers to see the same changing files for the first time. It was evaluated with: (i) unit tests for generative capabilities, (ii) an AI attacker (ARACNE) to assess realism and deception, (iii) human attackers to assess its deceptive capability, and (iv) an Internet deployment to evaluate deception in real-world attacks. In unit test results, AdvancedShelLM achieved a pass rate of up to 99.02%. The AI attacker ARACNE had issues making a decision if the system is honeypot or not, but showed slight bias towards saying honeypot, even for a real Ubuntu shell. With human attackers, AdvancedShelLM deceived more humans than Cowrie, but had similar results as shelLM. The Internet deployment showed concrete evidence that the output of AdvancedShelLM can influence the behaviour of real-life attackers.

### 方法论：核心思想与技术细节

**核心思想**：采用多智能体（Multi-Agent）多LLM架构，将Shell响应生成与验证分离，通过外部持久化文件系统实现状态一致性。

- **Worker智能体**：负责生成每条命令的Shell输出（草稿）。它接收系统提示（含当前文件系统JSON）和完整会话历史。
- **Manager智能体**：审查Worker的草稿，决定是否通过（APPROVED）或拒绝（REPROMPT + 修正指令）。被拒绝的草稿不加入会话历史，Worker最多重试3次。
- **文件系统状态**：以JSON格式编码存储（路径、类型、大小、时间戳、权限等）。Manager在每轮通过后输出文件系统补丁（ADD/REMOVE），更新外部存储，而非依赖LLM记忆。
- **时间感知**：Manager的真实提示中包含当前UTC时间，确保`date`、`uptime`等命令输出正确。
- **bash_history同步**：双向同步——启动时读取预置历史，关闭时将本次会话命令追加至历史记录，实现跨会话连贯。

**流程**：攻击者命令 → Worker生成草稿 → Manager审查+状态更新 → 输出给攻击者。

### 论文主要结论与发现

1. **生成正确性显著提升**：单元测试最高通过率99.02%（单会话98.04%），远超ShelLM（78.43%/83.33%）。本地模型gpt-oss-120B在某些配置下表现最佳。
2. **AI攻击者难以区分**：ARACNE对AdvancedShelLM、Cowrie、Ubuntu的蜜罐判定率接近（45% vs 47-53% vs 40-45%），且大部分结果不确定，说明系统未显著暴露。
3. **人类欺骗性有限**：71.4%的参与者认为AdvancedShelLM是蜜罐（与Ubuntu的70%相近），仅28.6%认为真实。Cowrie被100%识别。参与者普遍猜疑（因实验背景），但AdvancedShelLM仍有两名参与者判断为真实系统。
4. **互联网部署证实影响**：发现22个命令（0.0067%）显示LLM输出中的路径/URL等被后续攻击命令复用，证明蜜罐输出能改变攻击者行为。

### 优点

- **架构创新**：Worker-Manager多智能体分离生成与验证，有效减少不一致输出。
- **状态显式化**：外部JSON文件系统 + 跨会话历史同步，解决了LLM记忆不可靠的核心问题。
- **时间感知**：Manager注入真实时间，避免日期/uptime等命令错误。
- **评估全面**：单元测试、AI探测、人类盲测、真实部署四维度评估，覆盖生成正确性、自适应攻击、人类感知和实际影响。
- **开源实现**：代码、评估数据和脚本已公开，可复现。

### 不足与局限

- **延迟问题**：三重重试导致响应延迟模式可被察觉（人类反馈中“响应时间相同”）。

- **模型覆盖有限**：仅测试两个LLM，未探索更小/更快的替代方案或微调技术。
